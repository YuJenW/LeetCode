# 1732. Find the Highest Altitude 解題說明

## 1. 題目說明

有一位騎士正在騎車旅行。  
整趟旅程共有：

```text
n + 1 個地點
```

騎士從第 `0` 個地點出發，而且起始海拔為：

```text
0
```

題目給你一個整數陣列：

```text
gain
```

其中：

```text
gain[i]
```

代表：

```text
從第 i 個地點到第 i+1 個地點的海拔變化量
```

---

## 2. 題目要你找什麼？

請根據每一段的海拔變化，算出騎士在每個地點的實際海拔，並回傳：

```text
整趟旅途中出現過的最高海拔
```

---

## 3. 題目範例

### Example 1

```text
Input:
gain = [-5,1,5,0,-7]
```

起點海拔：

```text
0
```

依序計算：

```text
point 0: 0
point 1: 0 + (-5) = -5
point 2: -5 + 1 = -4
point 3: -4 + 5 = 1
point 4: 1 + 0 = 1
point 5: 1 + (-7) = -6
```

所以所有海拔為：

```text
[0, -5, -4, 1, 1, -6]
```

最高海拔是：

```text
1
```

因此輸出：

```text
1
```

---

### Example 2

```text
Input:
gain = [-4,-3,-2,-1,4,3,2]
```

海拔變化：

```text
point 0: 0
point 1: 0 - 4 = -4
point 2: -4 - 3 = -7
point 3: -7 - 2 = -9
point 4: -9 - 1 = -10
point 5: -10 + 4 = -6
point 6: -6 + 3 = -3
point 7: -3 + 2 = -1
```

所以所有海拔為：

```text
[0, -4, -7, -9, -10, -6, -3, -1]
```

最高海拔是：

```text
0
```

因為一開始的起點高度就是最高。

---

## 4. 這題的本質

這題本質上是在做：

```text
Prefix Sum，前綴和
```

因為每一個地點的海拔，都是：

```text
前面所有 gain 累積後的結果
```

---

## 5. 需要的知識點

這題主要需要理解：

```text
1. Array 陣列走訪
2. Prefix Sum 前綴和
3. 累積目前海拔
4. 同步更新最大值
5. 初始海拔 0 也要納入比較
```

---

## 6. 為什麼這題可以用 Prefix Sum？

假設：

```text
gain = [-5, 1, 5, 0, -7]
```

每一步的 altitude 都等於：

```text
前一個 altitude + 這一步的 gain
```

也就是：

```text
currentAltitude += gain[i]
```

這就是前綴和的概念。

---

## 7. 解題核心想法

我們只需要兩個變數：

```text
currentAltitude：目前海拔
highestAltitude：目前看過的最高海拔
```

---

### 一開始

```text
currentAltitude = 0
highestAltitude = 0
```

因為起點海拔就是 0。

---

### 每讀到一個 gain[i]

做兩件事：

```text
1. 更新目前海拔
2. 更新最高海拔
```

---

## 8. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    int largestAltitude(vector<int>& gain) {
        int currentAltitude = 0;
        int highestAltitude = 0;

        for (int change : gain) {
            currentAltitude += change;
            highestAltitude = max(highestAltitude, currentAltitude);
        }

        return highestAltitude;
    }
};
```

## 10. 程式逐行說明

### 10.1 目前海拔

```cpp
int currentAltitude = 0;
```

騎士從海拔 `0` 出發，所以目前高度一開始是：

```text
0
```

---

### 10.2 最高海拔

```cpp
int highestAltitude = 0;
```

一開始最高海拔也先設為：

```text
0
```

這很重要，因為有可能整趟路都在下降。  
例如 Example 2 的最高點就是起點 `0`。

---

### 10.3 走訪每個 gain

```cpp
for (int change : gain) {
```

這是 C++ 的 range-based for。

意思是：

```text
依序把 gain 裡的每個元素取出來，命名為 change
```

---

### 10.4 更新目前海拔

```cpp
currentAltitude += change;
```

如果目前海拔是 `-5`，這次 `change = 1`：

```text
currentAltitude = -5 + 1 = -4
```

---

### 10.5 更新目前最高海拔

```cpp
highestAltitude = max(highestAltitude, currentAltitude);
```

如果目前海拔比之前記錄的最高海拔還高，就更新。

---

### 10.6 回傳答案

```cpp
return highestAltitude;
```

最後回傳整趟旅程中最高的 altitude。

1

## 12. 為什麼不用另外建立 altitude 陣列？

你也可以先建立：

```text
altitudes = [0, -5, -4, 1, 1, -6]
```

再找最大值。

但其實沒有必要。

因為題目只問：

```text
最高海拔是多少
```

不需要保留所有海拔。

所以我們只要：

```text
邊累積，邊更新最大值
```

即可。

## 13. 複雜度分析

假設：

```text
n = gain.length
```

### 時間複雜度

```text
O(n)
```

因為每個元素只走訪一次。

---

### 空間複雜度

```text
O(1)
```

只使用兩個額外變數：

```text
currentAltitude
highestAltitude
```

---

## 14. Constraints 是否需要特別注意？

題目限制：

```text
1 <= n <= 100
-100 <= gain[i] <= 100
```

最大可能累積高度：

```text
100 × 100 = 10000
```

最小可能累積高度：

```text
100 × (-100) = -10000
```

都遠在 `int` 可表示範圍內，所以：

```cpp
int
```

完全足夠。

## 15. 面試講法

### 中文版

這題可以看成前綴和問題。  
我用 `currentAltitude` 累積目前海拔，並用 `highestAltitude` 紀錄目前出現過的最大值。  
騎士起點高度為 0，因此初始最大高度也要設為 0。  
每讀入一個 `gain[i]`，就更新目前高度，再更新最大高度。  
時間複雜度為 `O(n)`，空間複雜度為 `O(1)`。

---

### English Version

This problem can be solved using a prefix sum idea.  
I maintain `currentAltitude` to represent the current elevation and `highestAltitude` to track the maximum elevation seen so far.  
Since the biker starts at altitude 0, I initialize the maximum altitude to 0.  
For each gain value, I update the current altitude and then update the maximum altitude.  
The time complexity is `O(n)` and the space complexity is `O(1)`.
