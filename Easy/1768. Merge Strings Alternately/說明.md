# 1768. Merge Strings Alternately 解題說明

## 1. 題目說明

給定兩個字串：

```text
word1
word2
```

請從 `word1` 開始，依照交錯順序把兩個字串合併。

也就是：

```text
先放 word1 的第 1 個字元
再放 word2 的第 1 個字元
再放 word1 的第 2 個字元
再放 word2 的第 2 個字元
...
```

如果其中一個字串比較長，剩下的字元直接接到結果最後面。

---

## 2. 題目範例

### Example 1

```text
Input:
word1 = "abc"
word2 = "pqr"

Output:
"apbqcr"
```

合併過程：

```text
word1:  a   b   c
word2:    p   q   r
merged: a p b q c r
```

---

### Example 2

```text
Input:
word1 = "ab"
word2 = "pqrs"

Output:
"apbqrs"
```

合併過程：

```text
word1:  a   b
word2:    p   q   r   s
merged: a p b q r s
```

因為 `word2` 比較長，所以剩下的：

```text
r, s
```

直接接到最後。

---

### Example 3

```text
Input:
word1 = "abcd"
word2 = "pq"

Output:
"apbqcd"
```

合併過程：

```text
word1:  a   b   c   d
word2:    p   q
merged: a p b q c d
```

因為 `word1` 比較長，所以剩下的：

```text
c, d
```

直接接到最後。

## 5. 解題核心想法

我們建立一個新的字串：

```cpp
string merged;
```

接著使用兩個 index：

```text
i 指向 word1
j 指向 word2
```

只要兩個字串還有字元，就依序加入：

```text
word1[i]
word2[j]
```

如果某個字串先用完，就把另一個字串剩下的字元接到最後。

---

## 6. 雙指標設計

```text
i：目前讀到 word1 的位置
j：目前讀到 word2 的位置
```

例如：

```text
word1 = "abc"
word2 = "pqr"
```

初始：

```text
i = 0
j = 0
merged = ""
```

## 7. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    string mergeAlternately(string word1, string word2) {
        string merged;
        int i = 0;
        int j = 0;

        while (i < word1.size() || j < word2.size()) {
            if (i < word1.size()) {
                merged += word1[i];
                i++;
            }

            if (j < word2.size()) {
                merged += word2[j];
                j++;
            }
        }

        return merged;
    }
};
```

## 12. 為什麼 while 用 `||` 而不是 `&&`？

如果寫成：

```cpp
while (i < word1.size() && j < word2.size())
```

意思是：

```text
兩個字串都還有字元時才繼續
```

一旦其中一個字串先結束，迴圈就停了。

例如：

```text
word1 = "ab"
word2 = "pqrs"
```

只會處理到：

```text
"apbq"
```

剩下的：

```text
"rs"
```

就漏掉了。

所以正確是：

```cpp
while (i < word1.size() || j < word2.size())
```

## 13. 複雜度分析

假設：

```text
m = word1.length
n = word2.length
```

### 時間複雜度

```text
O(m + n)
```

因為每個字元都只會被加入一次。

---

### 空間複雜度

```text
O(m + n)
```

因為需要建立新的 merged 字串來存答案。

## 14. 另一種更精簡寫法

也可以先處理兩個字串共同長度的部分，再把剩餘字串接上去。

```cpp
class Solution {
public:
    string mergeAlternately(string word1, string word2) {
        string merged;
        int n = min(word1.size(), word2.size());

        for (int i = 0; i < n; i++) {
            merged += word1[i];
            merged += word2[i];
        }

        merged += word1.substr(n);
        merged += word2.substr(n);

        return merged;
    }
};
```

## 15. 兩種寫法比較

| 方法 | 優點 | 缺點 |
|---|---|---|
| while + 雙指標 | 邏輯直觀，適合初學 | 程式稍長 |
| for + substr | 程式較短 | 需要理解 `substr` |

## 16. 面試講法

### 中文版

我使用雙指標分別走訪 `word1` 和 `word2`。  
只要任一字串還有剩餘字元，就繼續處理。  
每一輪先加入 `word1` 目前字元，再加入 `word2` 目前字元。  
如果其中一個字串先結束，另一個字串的剩餘字元就會自然被加到結果最後。  
整體時間複雜度是 `O(m+n)`，空間複雜度也是 `O(m+n)`。

---

### English Version

I use two pointers to traverse `word1` and `word2`.  
As long as either string still has remaining characters, I continue merging.  
In each iteration, I first append a character from `word1`, then one from `word2`.  
If one string becomes exhausted earlier, the remaining characters of the other string are appended automatically.  
The time complexity is `O(m+n)`, and the space complexity is `O(m+n)`.

---

## 18. 進一步最佳化版本：預先配置空間 + 直接寫入結果字串

前面的寫法使用：

```cpp
merged += word1[i];
merged += word2[j];
```

或：

```cpp
merged.push_back(...)
```

這些寫法都可以正確完成題目。

但如果想進一步減少不必要的操作，可以改成：

```text
1. 先知道答案字串的最終長度
2. 一開始直接建立好完整長度的字串
3. 用 index 直接把字元寫進指定位置
```

---

## 19. 最佳化後的 LeetCode 程式碼

```cpp
class Solution {
public:
    string mergeAlternately(string word1, string word2) {
        int m = word1.size();
        int n = word2.size();
        int common = min(m, n);

        string ans(m + n, '\0');
        int k = 0;

        for (int i = 0; i < common; i++) {
            ans[k++] = word1[i];
            ans[k++] = word2[i];
        }

        for (int i = common; i < m; i++) {
            ans[k++] = word1[i];
        }

        for (int i = common; i < n; i++) {
            ans[k++] = word2[i];
        }

        return ans;
    }
};
```

## 20. 程式設計邏輯

這個版本可以拆成四個步驟：

```text
1. 取得兩個字串長度
2. 找出兩者共同能交錯合併的長度
3. 直接建立好最終結果字串
4. 分三段把字元寫入答案
```

```text

n = word2 的長度
```

```text

```



因此可以直接先建立：

## 25. 為什麼這個版本可能更快？

這個版本理論上比：

```cpp
while (...) {
    if (...) {
        merged.push_back(...);
    }
    if (...) {
        merged.push_back(...);
    }
}
```

更有效率，主要有三個原因。

---

### 原因 1：一開始就配置好答案字串大小

這個版本使用：

```cpp
string ans(m + n, '\0');
```

答案字串長度一次就固定好。

也就是：

```text
一開始就拿到足夠的空間
```

---

#### 相比之下

如果使用：

```cpp
string merged;
merged.push_back(...);
```

字串會隨著內容變長。

雖然 `std::string` 會自動管理容量，但當容量不足時，可能需要：

```text
1. 重新配置更大的記憶體
2. 把原本的內容複製到新空間
```

這些操作會增加額外成本。

---

### 原因 2：直接用 index 寫入，避免 append 類操作

最佳化版本：

```cpp
ans[k++] = word1[i];
```

是直接把字元寫進已經存在的空間。

相比：

```cpp
merged.push_back(word1[i]);
```

或：

```cpp
merged += word1[i];
```

直接 index 寫入可以省去部分：

```text
字串尾端追加的管理流程
```

---

### 原因 3：主迴圈不需要每次檢查兩個字串是否結束

原本 while 寫法：

```cpp
while (i < word1.size() || j < word2.size()) {
    if (i < word1.size()) {
        ...
    }

    if (j < word2.size()) {
        ...
    }
}
```

每一次 loop 都要判斷：

```text
word1 還有沒有字元？
word2 還有沒有字元？
```

---

最佳化版本先計算：

```cpp
int common = min(m, n);
```

因此第一個 for loop 可以直接保證：

```text
在 common 範圍內，word1 和 word2 都一定有字元
```

所以這段：

```cpp
for (int i = 0; i < common; i++) {
    ans[k++] = word1[i];
    ans[k++] = word2[i];
}
```

不需要再寫：

```cpp
if (i < word1.size())
if (i < word2.size())
```

判斷更少，流程更單純。

---

## 26. 為什麼 3 個 for 不會比較慢？

看到 3 個 `for`，有些人會直覺以為：

```text
迴圈變多，所以比較慢
```

但這個想法不準確。

---

### 實際上，總共處理的字元數沒有增加

不管哪種寫法，每個字元都只會被處理一次。

假設：

```text
word1 長度 = m
word2 長度 = n
```

總共需要處理：

```text
m + n 個字元
```

---

### 三個 for 的總工作量仍然是 m + n

```cpp
for (int i = 0; i < common; i++)
```

處理交錯區段。

```cpp
for (int i = common; i < m; i++)
```

處理 word1 剩餘區段。

```cpp
for (int i = common; i < n; i++)
```

處理 word2 剩餘區段。

三段加起來，仍然只是：

```text
每個字元寫入一次
```

所以整體時間複雜度仍然是：

```text
O(m + n)
```

---

## 27. 速度差異的真正來源

這個版本不是因為：

```text
3 個 for 比 1 個 while 神奇地更快
```

而是因為它：

```text
1. 先配置好完整字串空間
2. 用 index 直接寫入
3. 把流程拆成「交錯區段」與「剩餘區段」
4. 減少迴圈內的條件判斷
```

因此在實作層面上，通常會更有效率。

---

## 28. 與前一版比較

| 寫法 | 特點 | 可能成本 |
|---|---|---|
| `while + +=` | 最直觀 | 可能多次擴容，且每輪判斷較多 |
| `reserve + push_back` | 已減少擴容 | 仍需 push_back append |
| `固定長度 ans + index 寫入` | 直接填值，判斷較少 | 程式稍微長一些 |

## 29. 這版是否一定在 LeetCode 顯示更快？

不一定。

因為這題限制：

```text

```

輸入非常小。

LeetCode 顯示的：

```text

```

常常會受到：

```text

```

影響。

所以這版的優勢主要是：

```text
在程式設計上更接近低額外成本的實作
```

而不是保證每次 submission 都一定比其他版本快。
