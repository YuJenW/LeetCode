# 1768. Merge Strings Alternately 解題說明

## 1. 題目說明

給定兩個字串：

```text
word1
word2
```

請從 `word1` 開始，依照交錯順序把兩個字串合併。

也就是：

```text
先放 word1 的第 1 個字元
再放 word2 的第 1 個字元
再放 word1 的第 2 個字元
再放 word2 的第 2 個字元
...
```

如果其中一個字串比較長，剩下的字元直接接到結果最後面。

---

## 2. 題目範例

### Example 1

```text
Input:
word1 = "abc"
word2 = "pqr"

Output:
"apbqcr"
```

合併過程：

```text
word1:  a   b   c
word2:    p   q   r
merged: a p b q c r
```

---

### Example 2

```text
Input:
word1 = "ab"
word2 = "pqrs"

Output:
"apbqrs"
```

合併過程：

```text
word1:  a   b
word2:    p   q   r   s
merged: a p b q r s
```

因為 `word2` 比較長，所以剩下的：

```text
r, s
```

直接接到最後。

---

### Example 3

```text
Input:
word1 = "abcd"
word2 = "pq"

Output:
"apbqcd"
```

合併過程：

```text
word1:  a   b   c   d
word2:    p   q
merged: a p b q c d
```

因為 `word1` 比較長，所以剩下的：

```text
c, d
```

直接接到最後。

## 5. 解題核心想法

我們建立一個新的字串：

```cpp
string merged;
```

接著使用兩個 index：

```text
i 指向 word1
j 指向 word2
```

只要兩個字串還有字元，就依序加入：

```text
word1[i]
word2[j]
```

如果某個字串先用完，就把另一個字串剩下的字元接到最後。

---

## 6. 雙指標設計

```text
i：目前讀到 word1 的位置
j：目前讀到 word2 的位置
```

例如：

```text
word1 = "abc"
word2 = "pqr"
```

初始：

```text
i = 0
j = 0
merged = ""
```

## 7. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    string mergeAlternately(string word1, string word2) {
        string merged;
        int i = 0;
        int j = 0;

        while (i < word1.size() || j < word2.size()) {
            if (i < word1.size()) {
                merged += word1[i];
                i++;
            }

            if (j < word2.size()) {
                merged += word2[j];
                j++;
            }
        }

        return merged;
    }
};
```

```cpp

```

分別表示：

```text
i：word1 目前讀到的位置

```

### 9.3 while 條件

```cpp
while (i < word1.size() || j < word2.size())
```

意思是：

```text
只要 word1 還有字元，
或 word2 還有字元，
就繼續合併。
```

這裡使用的是：

```text
||
```

也就是邏輯 OR。

---

### 9.4 如果 word1 還有字元

```cpp
if (i < word1.size()) {
    merged += word1[i];
    i++;
}
```

意思是：

```text
如果 word1 還沒讀完，
就把目前的字元加入 merged，
然後 i 往後移一格。
```

---

### 9.5 如果 word2 還有字元

```cpp
if (j < word2.size()) {
    merged += word2[j];
    j++;
}
```

意思是：

```text
如果 word2 還沒讀完，
就把目前的字元加入 merged，
然後 j 往後移一格。
```

---

### 9.6 回傳答案

```cpp
return merged;
```

回傳最後合併完成的字串。

## 11. 長度不同時怎麼處理？

### Example

```text
word1 = "ab"
word2 = "pqrs"
```

---

### 前兩輪交錯合併

```text
a + p -> "ap"
b + q -> "apbq"
```

此時：

```text
word1 已經用完
word2 還剩 "rs"
```

因為 while 條件使用：

```cpp
i < word1.size() || j < word2.size()
```

所以迴圈會繼續。

接著只會執行：

```cpp
if (j < word2.size())
```

把剩下的：

```text
r
s
```

加到最後。

結果：

```text
"apbqrs"
```

---

## 12. 為什麼 while 用 `||` 而不是 `&&`？

如果寫成：

```cpp
while (i < word1.size() && j < word2.size())
```

意思是：

```text
兩個字串都還有字元時才繼續
```

一旦其中一個字串先結束，迴圈就停了。

例如：

```text
word1 = "ab"
word2 = "pqrs"
```

只會處理到：

```text
"apbq"
```

剩下的：

```text
"rs"
```

就漏掉了。

所以正確是：

```cpp
while (i < word1.size() || j < word2.size())
```

## 13. 複雜度分析

假設：

```text
m = word1.length
n = word2.length
```

### 時間複雜度

```text
O(m + n)
```

因為每個字元都只會被加入一次。

---

### 空間複雜度

```text
O(m + n)
```

因為需要建立新的 merged 字串來存答案。

---

## 14. 另一種更精簡寫法

也可以先處理兩個字串共同長度的部分，再把剩餘字串接上去。

```cpp
class Solution {
public:
    string mergeAlternately(string word1, string word2) {
        string merged;
        int n = min(word1.size(), word2.size());

        for (int i = 0; i < n; i++) {
            merged += word1[i];
            merged += word2[i];
        }

        merged += word1.substr(n);
        merged += word2.substr(n);

        return merged;
    }
};
```

## 15. 兩種寫法比較

| 方法 | 優點 | 缺點 |
|---|---|---|
| while + 雙指標 | 邏輯直觀，適合初學 | 程式稍長 |
| for + substr | 程式較短 | 需要理解 `substr` |

初學建議優先掌握：

```text
while + 雙指標
```

## 16. 面試講法

### 中文版

我使用雙指標分別走訪 `word1` 和 `word2`。  
只要任一字串還有剩餘字元，就繼續處理。  
每一輪先加入 `word1` 目前字元，再加入 `word2` 目前字元。  
如果其中一個字串先結束，另一個字串的剩餘字元就會自然被加到結果最後。  
整體時間複雜度是 `O(m+n)`，空間複雜度也是 `O(m+n)`。

---

### English Version

I use two pointers to traverse `word1` and `word2`.  
As long as either string still has remaining characters, I continue merging.  
In each iteration, I first append a character from `word1`, then one from `word2`.  
If one string becomes exhausted earlier, the remaining characters of the other string are appended automatically.  
The time complexity is `O(m+n)`, and the space complexity is `O(m+n)`.
