# 338. Counting Bits 解題說明

## 1. 題目說明

給定一個整數：

```text
n
```

請回傳一個長度為：

```text
n + 1
```

的陣列 `ans`。

其中：

```text
ans[i]
```

代表數字 `i` 的二進位表示中，有幾個 `1`。

---

## 2. 題目範例

### Example 1

```text
Input:
n = 2

Output:
[0,1,1]
```

原因：

```text
0 -> 0   有 0 個 1
1 -> 1   有 1 個 1
2 -> 10  有 1 個 1
```

所以：

```text
ans = [0,1,1]
```

---

### Example 2

```text
Input:
n = 5

Output:
[0,1,1,2,1,2]
```

原因：

```text
0 -> 0    有 0 個 1
1 -> 1    有 1 個 1
2 -> 10   有 1 個 1
3 -> 11   有 2 個 1
4 -> 100  有 1 個 1
5 -> 101  有 2 個 1
```

所以：

```text
ans = [0,1,1,2,1,2]
```

---

## 3. 題目要求

題目 follow-up 希望：

```text
1. 時間複雜度 O(n)
2. 最好只掃一次
3. 不使用內建函式，例如 __builtin_popcount
```

所以不建議直接對每個數字都重新計算二進位有幾個 `1`。

---

## 4. 需要的知識點

這題主要需要理解：

```text
1. Binary 二進位表示
2. Bit operation 位元運算
3. Dynamic Programming 動態規劃
4. 右移運算 >>
5. AND 運算 &
6. 奇偶數與二進位最低位元的關係
```

---

# 第一種想法：暴力法

## 5. 暴力法怎麼做？

對每個數字 `i`，都去計算它的二進位裡有幾個 `1`。

例如：

```cpp
int countOnes(int x) {
    int count = 0;

    while (x > 0) {
        if (x % 2 == 1) {
            count++;
        }
        x /= 2;
    }

    return count;
}
```

然後對：

```text
0 ~ n
```

全部做一次。

---

## 6. 暴力法問題

每個數字都要花：

```text
O(log n)
```

去拆二進位。

總共有 `n + 1` 個數字，所以總時間是：

```text
O(n log n)
```

但題目希望可以做到：

```text
O(n)
```

所以需要更好的方法。

---

# 第二種想法：DP + 右移

## 7. 核心觀察

一個數字 `i` 的二進位，可以看成：

```text
i 去掉最後一個 bit
+
最後一個 bit
```

---

例如：

```text
i = 5
```

二進位是：

```text
101
```

如果把最後一個 bit 拿掉：

```text
10
```

這就是：

```text
5 >> 1 = 2
```

而最後一個 bit 是：

```text
5 & 1 = 1
```

所以：

```text
5 的 1 的數量
=
2 的 1 的數量 + 1
```

也就是：

```text
ans[5] = ans[2] + 1
```

---

## 8. 公式

```cpp
ans[i] = ans[i >> 1] + (i & 1);
```

這是這題最推薦的公式。

---

## 9. 公式拆解

### `i >> 1`

```cpp
i >> 1
```

意思是：

```text
把 i 的二進位往右移一位
```

效果等同於：

```text
i / 2
```

但這是位元層級操作。

---

### `i & 1`

```cpp
i & 1
```

用來判斷：

```text
i 的最後一個 bit 是不是 1
```

也就是判斷 `i` 是奇數還是偶數。

---

## 10. `i & 1` 的結果

| i | binary | i & 1 |
|---:|---|---:|
| 0 | 0 | 0 |
| 1 | 1 | 1 |
| 2 | 10 | 0 |
| 3 | 11 | 1 |
| 4 | 100 | 0 |
| 5 | 101 | 1 |

所以：

```text
偶數 i & 1 = 0
奇數 i & 1 = 1
```

---

## 11. 用 n = 5 完整算一次

公式：

```cpp
ans[i] = ans[i >> 1] + (i & 1);
```

---

### i = 0

```text
ans[0] = 0
```

---

### i = 1

```text
1 >> 1 = 0
1 & 1 = 1

ans[1] = ans[0] + 1 = 1
```

---

### i = 2

```text
2 >> 1 = 1
2 & 1 = 0

ans[2] = ans[1] + 0 = 1
```

---

### i = 3

```text
3 >> 1 = 1
3 & 1 = 1

ans[3] = ans[1] + 1 = 2
```

---

### i = 4

```text
4 >> 1 = 2
4 & 1 = 0

ans[4] = ans[2] + 0 = 1
```

---

### i = 5

```text
5 >> 1 = 2
5 & 1 = 1

ans[5] = ans[2] + 1 = 2
```

---

最後：

```text
ans = [0,1,1,2,1,2]
```

## 12. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    vector<int> countBits(int n) {
        vector<int> ans(n + 1, 0);

        for (int i = 1; i <= n; i++) {
            ans[i] = ans[i >> 1] + (i & 1);
        }

        return ans;
    }
};
```

## 14. 程式逐行說明

### 建立答案陣列

```cpp
vector<int> ans(n + 1, 0);
```

因為要回傳：

```text
0 ~ n
```

每個數字的 bit count，所以長度是：

```text
n + 1
```

一開始全部設成 `0`。

---

### 為什麼從 i = 1 開始？

```cpp
for (int i = 1; i <= n; i++)
```

因為：

```text
ans[0] = 0
```

是已知的。

`0` 的二進位是：

```text
0
```

所以它有：

```text
0 個 1
```

---

### DP 更新公式

```cpp
ans[i] = ans[i >> 1] + (i & 1);
```

意思是：

```text
i 的 1 的數量
=
i 去掉最後一個 bit 後的 1 的數量
+
i 最後一個 bit 是否為 1
```

---

## 15. 為什麼這是 DP？

因為：

```text
ans[i] 會用到之前已經算好的 ans[i >> 1]
```

而：

```text
i >> 1 < i
```

所以在從小到大計算時，`ans[i >> 1]` 一定已經算好了。

---

## 16. 複雜度分析

假設輸入是：

```text
n
```

### 時間複雜度

```text
O(n)
```

因為每個 `i` 都只計算一次。

---

### 空間複雜度

```text
O(n)
```

因為題目要求回傳一個長度為 `n + 1` 的陣列。

如果不算輸出陣列，額外空間是：

```text
O(1)
```

# 第三種想法：DP + 最低有效位元

## 17. 另一個常見公式

還有另一個很常見的公式：

```cpp
ans[i] = ans[i & (i - 1)] + 1;
```

---

## 18. `i & (i - 1)` 是什麼？

```cpp
i & (i - 1)
```

可以把：

```text
i 的二進位中最右邊的一個 1 消掉
```

---

### 例子

```text
i = 12
binary = 1100
```

```text
i - 1 = 11
binary = 1011
```

```text
i & (i - 1)
= 1100
& 1011
= 1000
```

也就是把最右邊的 `1` 消掉。

---

## 19. 所以公式代表什麼？

```cpp
ans[i] = ans[i & (i - 1)] + 1;
```

意思是：

```text
i 的 1 的數量
=
刪掉最右邊一個 1 之後的數字的 1 的數量
+
被刪掉的那一個 1
```

---

## 20. LeetCode 程式碼：最低位元版本

```cpp
class Solution {
public:
    vector<int> countBits(int n) {
        vector<int> ans(n + 1, 0);

        for (int i = 1; i <= n; i++) {
            ans[i] = ans[i & (i - 1)] + 1;
        }

        return ans;
    }
};
```

---

## 21. 兩種 DP 公式比較

| 方法 | 公式 | 意思 | 推薦程度 |
|---|---|---|---|
| 右移法 | `ans[i] = ans[i >> 1] + (i & 1)` | 去掉最後一個 bit，再看最後 bit 是否為 1 | 最直觀 |
| 消最低位 1 | `ans[i] = ans[i & (i - 1)] + 1` | 消掉最右邊的一個 1 | 位元技巧更強 |

---

## 22. 哪個比較推薦？

如果是第一次學這題，推薦：

```text
ans[i] = ans[i >> 1] + (i & 1)
```

因為它比較容易從二進位結構理解。

如果想練 bit manipulation，可以再學：

```text
ans[i] = ans[i & (i - 1)] + 1
```

---

## 23. 面試講法

### 中文版

這題可以用動態規劃。  
對於每個數字 `i`，它的二進位可以拆成「去掉最後一位」和「最後一位」。  
`i >> 1` 等於去掉最後一個 bit，而 `i & 1` 可以判斷最後一個 bit 是否為 1。  
因此可以得到公式：`ans[i] = ans[i >> 1] + (i & 1)`。  
這樣每個數字只需 O(1) 時間計算，所以總時間是 O(n)。

---

### English Version

This problem can be solved using dynamic programming.  
For each number `i`, its binary representation can be divided into two parts: all bits except the last one, and the last bit.  
`i >> 1` removes the last bit, and `i & 1` tells whether the last bit is 1.  
Therefore, we can use the relation `ans[i] = ans[i >> 1] + (i & 1)`.  
Each value is computed in O(1), so the total time complexity is O(n).
