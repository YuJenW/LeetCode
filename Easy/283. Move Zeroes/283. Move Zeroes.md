# 283. Move Zeroes 解題說明

## 1. 題目說明

給定一個整數陣列 `nums`，請將所有的 `0` 移到陣列最後面。

同時必須滿足兩個條件：

```text
1. 非 0 元素原本的相對順序不能改變
2. 必須 in-place 原地修改，不能建立新的陣列
```

---

## 2. 題目範例

### Example 1

```text
Input: nums = [0,1,0,3,12]
Output: [1,3,12,0,0]
```

原本的非 0 元素順序是：

```text
1, 3, 12
```

移動後仍然維持：

```text
1, 3, 12
```

只是所有的 `0` 被移到最後。

---

### Example 2

```text
Input: nums = [0]
Output: [0]
```

陣列中只有一個 `0`，不需要做任何改動。

---

## 3. 這題適合使用什麼方法？

這題適合使用：

```text
Two Pointers 雙指標
```

原因是我們需要同時做到：

```text
1. 從左到右掃描整個陣列
2. 記錄下一個非 0 元素應該放在哪個位置
```

---

## 4. 需要的知識點

這題主要需要理解：

```text
1. Array 陣列走訪
2. In-place 原地修改
3. Two Pointers 雙指標
4. swap 交換元素
5. 如何維持元素的相對順序
6. 如何減少不必要的操作次數
```

---

## 5. 直觀想法

題目要我們把所有 `0` 移到最後。

換個角度想：

```text
其實就是把所有非 0 元素，依照原本順序依序放到前面。
```

例如：

```text
nums = [0,1,0,3,12]
```

非 0 元素依序是：

```text
1, 3, 12
```

所以結果應該是：

```text
[1,3,12,0,0]
```

---

## 6. 雙指標設計

我們使用兩個指標：

```text
i    ：負責掃描陣列中的每一個位置
slow ：表示下一個非 0 元素應該放到的位置
```

---

### 初始狀態

```text
nums = [0,1,0,3,12]
slow = 0
```

意思是：

```text
下一個找到的非 0 元素，應該放到 index 0。
```

---

## 7. 解法核心邏輯

當我們從左到右掃描陣列：

- 如果 `nums[i] == 0`
  - 跳過
- 如果 `nums[i] != 0`
  - 這個數字要被移到前面
  - 放到 `slow` 指向的位置
  - 接著 `slow++`

---

## 8. 為什麼用 swap？

這題 follow-up 問：

```text
Could you minimize the total number of operations done?
```

也就是：

```text
能不能減少總操作次數？
```

如果使用覆寫法：

```cpp
nums[slow] = nums[i];
```

之後還要再把後面補成 0。

雖然正確，但可能多做一些寫入。

因此更好的做法是：

```cpp
swap(nums[i], nums[slow]);
```

這樣當非 0 元素被移到前面時，原本前面的 0 會自然被交換到後面。

---

## 9. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int slow = 0;

        for (int i = 0; i < nums.size(); i++) {
            if (nums[i] != 0) {
                if (i != slow) {
                    swap(nums[i], nums[slow]);
                }
                slow++;
            }
        }
    }
};
```

## 11. 程式逐行說明

### 11.1 建立 slow 指標

```cpp
int slow = 0;
```

`slow` 代表：

```text
下一個非 0 元素應該放到哪個位置
```

一開始還沒有放任何非 0 元素，所以從 index 0 開始。

---

### 11.2 掃描整個陣列

```cpp
for (int i = 0; i < nums.size(); i++) {
```

`i` 會從頭掃到尾，用來檢查每一個元素。

---

### 11.3 找到非 0 元素

```cpp
if (nums[i] != 0) {
```

如果目前元素不是 0，代表這個元素應該被保留到前面。

---

### 11.4 判斷是否需要交換

```cpp
if (i != slow) {
    swap(nums[i], nums[slow]);
}
```

如果：

```text
i == slow
```

代表這個非 0 元素本來就在正確位置，不需要交換。

如果：

```text
i != slow
```

代表目前非 0 元素在比較後面，需要交換到 `slow` 位置。

---

### 11.5 slow 往後移

```cpp
slow++;
```

因為已經成功放入一個非 0 元素，所以下一個非 0 元素應該放在下一格。

## 14. 為什麼相對順序不會改變？

因為我們是：

```text
從左到右依序掃描非 0 元素
```

找到的順序一定是：

```text
1 -> 3 -> 12
```

接著依序放進前面的位置：

```text
index 0 放 1
index 1 放 3
index 2 放 12
```

所以原本的相對順序會被保留。

---

## 15. 為什麼這是 in-place？

因為我們：

```text
直接在原本的 nums 陣列上交換元素
```

沒有建立另一個新的陣列。

因此符合：

```text
in-place without making a copy of the array
```

---

## 16. 為什麼這樣能減少操作次數？

因為加了這個判斷：

```cpp
if (i != slow)
```

如果非 0 元素本來就在正確位置，就不做交換。

---

### 例子

```text
nums = [1,2,3,0,0]
```

一開始：

```text
slow = 0
```

掃到：

```text
i = 0，nums[0] = 1
```

這時：

```text
i == slow
```

所以不交換。

同理：

```text
2 和 3 也不需要交換
```

這樣就避免了不必要的操作。

假設陣列長度是 `n`。

### 時間複雜度

```text

### 空間複雜度

```

## 18. 覆寫法 vs swap 法

### 覆寫法

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int slow = 0;

        for (int i = 0; i < nums.size(); i++) {
            if (nums[i] != 0) {
                nums[slow] = nums[i];
                slow++;
            }
        }

        while (slow < nums.size()) {
            nums[slow] = 0;
            slow++;
        }
    }
};
```

---

### swap 法

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int slow = 0;

        for (int i = 0; i < nums.size(); i++) {
            if (nums[i] != 0) {
                if (i != slow) {
                    swap(nums[i], nums[slow]);
                }
                slow++;
            }
        }
    }
};
```

---

### 比較

| 方法 | 優點 | 缺點 |
|---|---|---|
| 覆寫法 | 邏輯直觀 | 需要後續補 0 |
| swap 法 | 操作較少，更符合 follow-up | 需要理解交換邏輯 |

## 20. 面試講法

### 中文版

我使用雙指標。  
`i` 負責從左到右掃描整個陣列，`slow` 表示下一個非 0 元素應該放置的位置。  
當我遇到非 0 元素時，如果 `i != slow`，就把 `nums[i]` 和 `nums[slow]` 交換，接著讓 `slow` 往後移。  
這樣可以保持非 0 元素的相對順序，並在原地完成操作。  
時間複雜度是 `O(n)`，空間複雜度是 `O(1)`。

---

### English Version

I use two pointers.  
`i` scans the entire array from left to right, and `slow` indicates the next position where a non-zero element should be placed.  
When I encounter a non-zero element, I swap it with `nums[slow]` only if `i != slow`, then move `slow` forward.  
This preserves the relative order of non-zero elements and performs the operation in-place.  
The time complexity is `O(n)` and the space complexity is `O(1)`.
