# 9. Palindrome Number 解題說明

## 1. 題目說明

給定一個整數 `x`，判斷它是不是回文數。

如果一個數字從左到右讀，和從右到左讀都一樣，就是回文數。

---

## 2. 題目範例

### Example 1

```text
Input: x = 121
Output: true
```

原因：

```text
121 從左到右是 121
121 從右到左也是 121
```

所以是回文數。

---

### Example 2

```text
Input: x = -121
Output: false
```

原因：

```text
-121 反過來會變成 121-
```

這不是合法相同的數字，所以不是回文數。

---

### Example 3

```text
Input: x = 10
Output: false
```

原因：

```text
10 反過來是 01
```

也就是 `1`，和原本的 `10` 不一樣。

所以不是回文數。

---

## 3. 需要的知識點

這題需要理解幾個基本觀念：

```text
1. 回文 Palindrome 的定義
2. 負數不可能是回文數
3. 取個位數 %
4. 去掉個位數 /
5. 整數反轉
6. overflow 風險
```

---

## 4. 最直覺解法：轉成字串

最簡單的方法是把整數轉成字串，再用雙指標判斷。

例如：

```text
x = 121
s = "121"
```

比較：

```text
s[0] 和 s[2]
s[1] 和 s[1]
```

如果全部相同，就是回文。

---

### 字串版本程式碼

```cpp
class Solution {
public:
    bool isPalindrome(int x) {
        string s = to_string(x);

        int left = 0;
        int right = s.size() - 1;

        while (left < right) {
            if (s[left] != s[right]) {
                return false;
            }

            left++;
            right--;
        }

        return true;
    }
};
```

---

### 字串版本優缺點

優點：

```text
簡單好理解
```

缺點：

```text
使用了額外字串空間
沒有滿足 follow-up 的要求
```

複雜度：

```text
Time Complexity: O(n)
Space Complexity: O(n)
```

其中 `n` 是數字的位數。

---

## 5. Follow-up：不轉字串怎麼做？

題目問：

```text
Could you solve it without converting the integer to a string?
```

也就是：

```text
能不能不要把整數轉成字串？
```

可以。

我們可以用數學方式把數字反轉，再比較是否相同。

## 6. 數學解法核心想法

假設：

```text
x = 121
```

我們可以把它反轉成：

```text
121
```

如果反轉後和原本相同，就是回文。

---

## 7. 如何取出數字的最後一位？

使用 `% 10`。

```cpp
int digit = x % 10;
```

例如：

```text
121 % 10 = 1
123 % 10 = 3
10 % 10 = 0
```

---

## 8. 如何移除數字的最後一位？

使用 `/ 10`。

```cpp
x = x / 10;
```

例如：

```text
121 / 10 = 12
12 / 10 = 1
1 / 10 = 0
```

---

## 9. 如何反轉整數？

假設：

```text
x = 121
reversed = 0
```

每次取出 x 的最後一位，加到 `reversed` 後面。

### 第 1 輪

```text
digit = 121 % 10 = 1
reversed = 0 * 10 + 1 = 1
x = 121 / 10 = 12
```

---

### 第 2 輪

```text
digit = 12 % 10 = 2
reversed = 1 * 10 + 2 = 12
x = 12 / 10 = 1
```

---

### 第 3 輪

```text
digit = 1 % 10 = 1
reversed = 12 * 10 + 1 = 121
x = 1 / 10 = 0
```

最後：

```text
reversed = 121
```

和原本相同，所以是回文。

---

## 10. 直接反轉整個數字的版本

```cpp
class Solution {
public:
    bool isPalindrome(int x) {
        if (x < 0) {
            return false;
        }

        long long original = x;
        long long reversed = 0;

        while (x > 0) {
            int digit = x % 10;
            reversed = reversed * 10 + digit;
            x = x / 10;
        }

        return original == reversed;
    }
};
```

---

## 11. 為什麼要先處理負數？

```cpp
if (x < 0) {
    return false;
}
```

負數一定不是回文。

例如：

```text
-121
```

從右到左會變成：

```text
121-
```

所以不可能是回文。

---

## 12. 為什麼使用 long long？

```cpp
long long reversed = 0;
```

因為如果直接反轉整個整數，有可能發生 overflow。

雖然這題只是判斷回文，但反轉過程中可能產生比 `int` 更大的值。

所以用 `long long` 比較安全。

---

## 13. 更好的數學解法：只反轉一半

其實不需要反轉整個數字。

原因是：

```text
只要前半段和後半段相同，就是回文。
```

例如：

```text
x = 1221
```

我們只需要把後半段反轉：

```text
前半段：12
後半段反轉：12
```

相同，所以是回文。

---

## 14. 為什麼只反轉一半比較好？

因為可以避免整數 overflow。

如果只反轉一半，`reversed` 不會變得太大。

這是 LeetCode 這題最推薦的解法。

---

## 15. 先排除不可能情況

### 負數

```cpp
x < 0
```

一定不是回文。

---

### 以 0 結尾但不是 0 的數字

例如：

```text
10
100
120
```

這些都不是回文。

原因：

```text
10 反過來是 01
100 反過來是 001
```

正常整數不會保留開頭的 0。

所以：

```cpp
if (x < 0 || (x % 10 == 0 && x != 0)) {
    return false;
}
```

---

## 16. 只反轉一半的想法

假設：

```text
x = 1221
```

一開始：

```text
x = 1221
reversed = 0
```

---

### 第 1 輪

```text
digit = 1221 % 10 = 1
reversed = 0 * 10 + 1 = 1
x = 1221 / 10 = 122
```

---

### 第 2 輪

```text
digit = 122 % 10 = 2
reversed = 1 * 10 + 2 = 12
x = 122 / 10 = 12
```

現在：

```text
x = 12
reversed = 12
```

代表前半段和後半段反轉後相同。

所以是回文。

---

## 17. 奇數位數怎麼辦？

假設：

```text
x = 121
```

---

### 第 1 輪

```text
digit = 121 % 10 = 1
reversed = 1
x = 12
```

---

### 第 2 輪

```text
digit = 12 % 10 = 2
reversed = 12
x = 1
```

這時：

```text
x = 1
reversed = 12
```

因為奇數位數中間的數字不影響回文。

例如：

```text
121
中間的 2 可以忽略
```

所以比較時要用：

```cpp
x == reversed / 10
```

因為：

```text
reversed = 12
reversed / 10 = 1
```

剛好和 `x` 相同。

---

## 18. LeetCode 推薦解法：不轉字串，只反轉一半

```cpp
class Solution {
public:
    bool isPalindrome(int x) {
        if (x < 0 || (x % 10 == 0 && x != 0)) {
            return false;
        }

        int reversed = 0;

        while (x > reversed) {
            int digit = x % 10;
            reversed = reversed * 10 + digit;
            x = x / 10;
        }

        return x == reversed || x == reversed / 10;
    }
};
```

---

## 19. 程式逐行說明

### 排除負數與尾數為 0 的數字

```cpp
if (x < 0 || (x % 10 == 0 && x != 0)) {
    return false;
}
```

這裡排除兩種情況：

```text
1. 負數
2. 不是 0 但尾數是 0 的數字
```

例如：

```text
-121 -> false
10   -> false
100  -> false
```

但：

```text
0 -> true
```

所以要保留 `x != 0` 的判斷。

---

### 建立 reversed

```cpp
int reversed = 0;
```

`reversed` 用來存反轉後的後半段數字。

---

### 只反轉一半

```cpp
while (x > reversed) {
```

這個條件表示：

```text
當原本剩下的前半段還比反轉後的後半段大時，就繼續拆數字。
```

當 `reversed` 已經大於或等於 `x` 時，代表已經處理到一半了。

---

### 取最後一位

```cpp
int digit = x % 10;
```

取得 `x` 的最後一位數字。

---

### 接到 reversed 後面

```cpp
reversed = reversed * 10 + digit;
```

把剛取出的 digit 接到 `reversed` 的後面。

---

### 移除 x 的最後一位

```cpp
x = x / 10;
```

把 `x` 的最後一位移除。

---

### 最後比較

```cpp
return x == reversed || x == reversed / 10;
```

分成兩種情況：

---

#### 偶數位數

例如：

```text
1221
```

最後會變成：

```text
x = 12
reversed = 12
```

所以：

```cpp
x == reversed
```

成立。

---

#### 奇數位數

例如：

```text
121
```

最後會變成：

```text
x = 1
reversed = 12
```

中間的 `2` 不需要比較，所以：

```cpp
x == reversed / 10
```

成立。

---

## 20. 複雜度分析

假設數字有 `d` 位數。

### Time Complexity

```text
O(d)
```

因為每次都去掉一位數。

如果只反轉一半，實際上是：

```text
O(d / 2)
```

但 Big-O 仍然是：

```text
O(d)
```

---

### Space Complexity

```text
O(1)
```

因為沒有使用字串或額外陣列，只使用幾個變數。

## 21. 面試講法

### 中文版

這題可以用字串雙指標解，但 follow-up 要求不能轉字串，所以我使用數學方式處理。  
我先排除負數和尾數為 0 但不等於 0 的數字。  
接著只反轉數字的後半段，避免整數 overflow。  
當反轉後的後半段大於或等於剩下的前半段時，就停止。  
最後如果是偶數位數，檢查 `x == reversed`；如果是奇數位數，忽略中間位數，檢查 `x == reversed / 10`。

---

### English Version

I can solve this by converting the integer to a string, but the follow-up asks for a solution without string conversion.  
So I use a mathematical approach and reverse only half of the number to avoid overflow.  
First, I reject negative numbers and numbers ending with zero except zero itself.  
Then I extract digits from the end of `x` and build the reversed half.  
Finally, for even-length numbers I compare `x == reversed`, and for odd-length numbers I compare `x == reversed / 10`.

## 23. String 版本 vs long long int 版本：哪個比較省記憶體？

在判斷 Palindrome Number 時，常見有兩種做法：

```text
1. 把整數轉成 string，再用雙指標比較
2. 不轉 string，直接用數學方式反轉整數
```

如果比較記憶體使用量：

```text
long long int 數學版本比較省記憶體
```

---

## 24. 為什麼 string 版本比較耗記憶體？

如果使用 string：

```cpp
string s = to_string(x);
```

假設：

```text
x = 123456789
```

會建立一個新的字串：

```text
"123456789"
```

這個字串需要額外空間來存每一個字元。

所以 string 版本的額外空間複雜度是：

```text
O(d)
```

其中 `d` 是數字的位數。

例如：

```text
123456789 有 9 位數，所以需要額外存 9 個字元左右。
```

---

## 25. 為什麼 long long int 版本比較省記憶體？

如果使用數學方式：

```cpp
long long original = x;
long long reversed = 0;
```

只需要幾個固定變數：

```text
original
reversed
digit
```

不管數字有幾位，變數數量都不會增加。

所以額外空間複雜度是：

```text
O(1)
```

也就是固定空間。

---

## 26. String 版本與 long long int 版本比較

| 方法 | 額外空間 | 說明 |
|---|---:|---|
| string 版本 | O(d) | 需要額外建立字串 |
| long long int 完整反轉版本 | O(1) | 只使用固定數量變數 |
| 只反轉一半版本 | O(1) | 不轉字串，也降低 overflow 風險 |

其中最推薦的是：

```text
只反轉一半版本
```

因為它：

```text
1. 不需要 string
2. 不需要額外陣列
3. 空間複雜度是 O(1)
4. 可以降低完整反轉時 overflow 的風險
```

## 27. C++ string 版本

```cpp
class Solution {
public:
    bool isPalindrome(int x) {
        string s = to_string(x);

        int left = 0;
        int right = s.size() - 1;

        while (left < right) {
            if (s[left] != s[right]) {
                return false;
            }

            left++;
            right--;
        }

        return true;
    }
};
```

### 特性

```text
Time Complexity: O(d)
Space Complexity: O(d)
```

其中 `d` 是數字位數。

優點：

```text
寫法簡單，容易理解。
```

缺點：

```text

```

```c
bool isPalindrome(int x) {

        right--;
    }

    return true;
}
```

### 特性

```text
Time Complexity: O(d)
```

因為使用了：
