# 443. String Compression 解題說明

## 1. 題目說明

給定一個字元陣列：

```text
chars
```

請將其中**連續重複出現的字元**進行壓縮。

壓縮規則如下：

- 如果某個字元只連續出現 `1` 次  
  → 只保留該字元
- 如果某個字元連續出現超過 `1` 次  
  → 保留該字元，後面接上出現次數

---

## 2. 題目範例

### Example 1

```text
Input:
chars = ["a","a","b","b","c","c","c"]
```

可分成三組：

```text
"aa", "bb", "ccc"
```

壓縮後：

```text
"a2b2c3"
```

所以：

```text
Output: 6
```

因為壓縮後有效長度是：

```text
6
```

---

### Example 2

```text
Input:
chars = ["a"]
```

只有一個字元，不能再壓縮。

```text
Output: 1
```

---

### Example 3

```text
Input:
chars = ["a","b","b","b","b","b","b","b","b","b","b","b","b"]
```

分組為：

```text
"a"
"bbbbbbbbbbbb"
```

第二組 `b` 有 `12` 個。

壓縮後：

```text
"ab12"
```

所以：

```text
Output: 4
```

---

## 3. 題目真正要求什麼？

這題有兩個很重要的限制：

### 限制 1：不能回傳新的字串

題目要求：

```text
壓縮結果要直接寫回原本的 chars 陣列中
```

不是另外建立：

```cpp
string s;
```

再回傳它。

---

### 限制 2：只能使用常數額外空間

題目要求：

```text
uses only constant extra space
```

也就是：

```text
Space Complexity: O(1)
```

不能額外建立另一個和輸入大小成比例的陣列。

---

## 4. 能應用的場景

這題本質是：

```text
Run-Length Encoding，RLE，連續重複資料壓縮
```

常見應用：

```text
1. 簡易字串壓縮
2. 影像中的游程編碼
3. 連續訊號資料壓縮
4. 將重複資料轉成「字元 + 次數」格式
```

例如：

```text
AAAAABBBCC
```

可以壓縮為：

```text
A5B3C2
```

---

## 5. 需要的知識點

這題主要需要理解：

```text
1. Array 原地修改
2. Two Pointers 雙指標
3. 連續區段分組
4. 數字轉成字元
5. 兩位數以上 count 的處理
6. write pointer 寫回原陣列
```

---

## 6. 核心解法：讀取指標 + 寫入指標

我們使用兩個主要位置：

```text
read  ：負責掃描原始 chars
write ：負責把壓縮後的結果寫回 chars
```

---

## 7. 設計邏輯

對每一組連續相同字元：

```text
1. 記住這一組的字元 ch
2. 計算這個字元連續出現幾次 count
3. 把 ch 寫回 chars[write]
4. 如果 count > 1：
   - 把 count 的每一位數字寫回 chars
5. 繼續處理下一組
```

---

## 8. 為什麼需要 write 指標？

因為原始字串會被壓縮成更短的內容。

例如：

```text
["a","a","b","b","c","c","c"]
```

壓縮成：

```text
["a","2","b","2","c","3"]
```

我們必須直接寫回原本 `chars` 裡。

所以：

```text
write 代表下一個壓縮結果要寫入的位置
```

---

## 9. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    int compress(vector<char>& chars) {
        int write = 0;
        int read = 0;
        int n = chars.size();

        while (read < n) {
            char currentChar = chars[read];
            int count = 0;

            // 計算目前字元連續出現幾次
            while (read < n && chars[read] == currentChar) {
                read++;
                count++;
            }

            // 先寫入字元本身
            chars[write++] = currentChar;

            // 如果出現超過一次，再寫入次數
            if (count > 1) {
                string countStr = to_string(count);

                for (char digit : countStr) {
                    chars[write++] = digit;
                }
            }
        }

        return write;
    }
};
```

---

## 10. 這個版本符合 constant extra space 嗎？

這裡要特別注意。

上述版本使用了：

```cpp
string countStr = to_string(count);
```

由於題目限制：

```text
chars.length <= 2000
```

所以 count 最多是 `2000`，數字字串長度最多只有：

```text
4
```

很多 LeetCode 解法會接受這種寫法，實務上額外空間非常小。

但如果要嚴格做到：

```text
完全 O(1) 額外空間，連短暫 string 都不建立
```

可以使用下面的版本。

## 11. 更嚴格的 O(1) 額外空間版本

```cpp
class Solution {
public:
    int compress(vector<char>& chars) {
        int write = 0;
        int read = 0;
        int n = chars.size();

        while (read < n) {
            char currentChar = chars[read];
            int count = 0;

            // 計算目前字元連續出現幾次
            while (read < n && chars[read] == currentChar) {
                read++;
                count++;
            }

            // 寫入字元
            chars[write++] = currentChar;

            // 如果 count > 1，將 count 拆成每一位 digit 寫入
            if (count > 1) {
                int start = write;

                // 先反向寫入個位、十位、百位...
                while (count > 0) {
                    chars[write++] = '0' + (count % 10); //ascii to char
                    count /= 10;
                }

                // 因為剛剛是反向寫入，所以需要反轉回來
                reverse(chars.begin() + start, chars.begin() + write);
            }
        }

        return write;
    }
};
```

---

## 12. 為什麼數字要反轉？

假設：

```text
count = 12
```

如果使用：

```cpp
count % 10
```

會先取得：

```text
2
```

再取得：

```text
1
```

所以寫入順序會是：

```text
"21"
```

但正確應該是：

```text
"12"
```

因此需要再反轉一次。

## 13. 程式逐行說明

以下以比較好理解的 `to_string()` 版本說明。

---

### 13.1 建立 write 與 read

```cpp
int write = 0;
int read = 0;
```

意思是：

```text
read  負責讀原始陣列
write 負責寫壓縮後結果
```

---

### 13.2 紀錄陣列長度

```cpp
int n = chars.size();
```

---

### 13.3 外層 while：逐組處理

```cpp
while (read < n) {
```

只要還沒掃描完整個 chars，就繼續。

---

### 13.4 記住目前這組的字元

```cpp
char currentChar = chars[read];
```

例如目前讀到：

```text
'a'
```

---

### 13.5 初始化計數

```cpp
int count = 0;
```

準備計算這一組連續出現幾次。

---

### 13.6 內層 while：數連續長度

```cpp
while (read < n && chars[read] == currentChar) {
    read++;
    count++;
}
```

例如：

```text
["a","a","a","b"]
```

目前 `currentChar = 'a'`

這個迴圈會跑三次，最後：

```text
count = 3
read 指到 'b'
```

---

### 13.7 寫入字元本身

```cpp
chars[write++] = currentChar;
```

例如：

```text
'a'
```

先寫回壓縮後陣列。

---

### 13.8 如果 count > 1，寫入數字

```cpp
if (count > 1) {
    string countStr = to_string(count);

    for (char digit : countStr) {
        chars[write++] = digit;
    }
}
```

例如：

```text
count = 12
```

會先轉成：

```text
"12"
```

再逐字寫入：

```text
'1'
'2'
```

---

### 13.9 回傳新長度

```cpp
return write;
```

`write` 最後停的位置，就是壓縮後的有效長度。

## 16. 複雜度分析

假設：

```text
n = chars.length
```

### 時間複雜度

```text
O(n)
```

原因：

```text
read 指標會從頭走到尾一次
每個字元最多被讀取一次
```

如果使用 `to_string()` 版本：

```text

```


## 18. 面試講法

### 中文版

我使用雙指標處理。  
`read` 用來掃描原始字元陣列，`write` 用來把壓縮後的結果寫回原陣列。  
每次先找出一組連續相同字元的長度 `count`，接著寫入字元本身；若 `count > 1`，再把次數的每一位數字依序寫回。  
因為整個過程直接修改原陣列，所以只需要常數額外空間。

---

### English Version

I use two pointers.  
`read` scans the original character array, and `write` stores the compressed result back into the same array.  
For each consecutive group, I count its length, write the character first, and if the count is greater than one, I write each digit of the count.  
Since the result is written in-place, the algorithm uses constant extra space.

---

## 19. 一句話總結

```text
這題的核心是：用 read 找出每一組連續字元，用 write 將壓縮結果原地寫回 chars。
```
