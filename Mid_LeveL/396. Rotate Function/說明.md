# 396. Rotate Function 解題說明

## 1. 題目說明

給定一個整數陣列 `nums`，長度為 `n`。

定義 `arrk` 為將 `nums` 向右旋轉 `k` 次後得到的陣列。

接著定義旋轉函數：

```text
F(k) = 0 * arrk[0] + 1 * arrk[1] + ... + (n - 1) * arrk[n - 1]
```

也就是：

```text
每個元素乘上它目前所在的 index，然後全部加總。
```

目標是找出：

```text
F(0), F(1), F(2), ..., F(n-1)
```

之中的最大值。

---

## 2. 題目範例

### Example

```text
nums = [4, 3, 2, 6]
```

---

### F(0)

沒有旋轉：

```text
arr0 = [4, 3, 2, 6]
```

計算：

```text
F(0) = 0*4 + 1*3 + 2*2 + 3*6
     = 0 + 3 + 4 + 18
     = 25
```

---

### F(1)

向右旋轉一次：

```text
arr1 = [6, 4, 3, 2]
```

計算：

```text
F(1) = 0*6 + 1*4 + 2*3 + 3*2
     = 0 + 4 + 6 + 6
     = 16
```

---

### F(2)

向右旋轉兩次：

```text
arr2 = [2, 6, 4, 3]
```

計算：

```text
F(2) = 0*2 + 1*6 + 2*4 + 3*3
     = 0 + 6 + 8 + 9
     = 23
```

---

### F(3)

向右旋轉三次：

```text
arr3 = [3, 2, 6, 4]
```

計算：

```text
F(3) = 0*3 + 1*2 + 2*6 + 3*4
     = 0 + 2 + 12 + 12
     = 26
```

所以最大值是：

```text
26
```

---

## 3. 這題不能用暴力法

最直覺的做法是：

```text
1. 對每一個 k 都真的旋轉一次陣列
2. 計算 F(k)
3. 取最大值
```

但這樣很慢。

因為：

```text
n 最大可以到 100000
```

如果每次旋轉都重新計算一次：

```text
F(0), F(1), ..., F(n-1)
```

每個都花 `O(n)`，總時間會是：

```text
O(n^2)
```

當 `n = 100000` 時，`O(n^2)` 會超時。

所以這題的重點是：

```text
找出 F(k) 和 F(k-1) 之間的關係
```

---

## 4. 需要的知識點

這題主要需要：

```text
1. 陣列旋轉的概念
2. 數學遞推公式
3. 前一次結果推下一次結果
4. 時間複雜度優化
```

---

## 5. 先觀察 F(0)

假設：

```text
nums = [4, 3, 2, 6]
n = 4
```

那：

```text
F(0) = 0*4 + 1*3 + 2*2 + 3*6
```

也就是：

```text
F(0) = 25
```

---

## 6. 總和 sum 的用途

先計算整個陣列總和：

```text
sum = nums[0] + nums[1] + ... + nums[n-1]
```

以範例來說：

```text
sum = 4 + 3 + 2 + 6 = 15
```

這個 `sum` 會用在遞推公式裡。

---

## 7. 推導 F(k) 的關係

### 原始陣列

```text
nums = [a0, a1, a2, ..., a(n-1)]
```

---

### F(0)

```text
F(0) = 0*a0 + 1*a1 + 2*a2 + ... + (n-1)*a(n-1)
```

---

### 向右旋轉一次後

```text
arr1 = [a(n-1), a0, a1, a2, ..., a(n-2)]
```

所以：

```text
F(1) = 0*a(n-1) + 1*a0 + 2*a1 + 3*a2 + ... + (n-1)*a(n-2)
```

---

## 8. 從 F(0) 推 F(1)

觀察旋轉一次後：

除了最後一個元素 `a(n-1)` 會跑到最前面，其餘元素的 index 都增加 1。

也就是：

```text
a0     index 從 0 變 1
a1     index 從 1 變 2
a2     index 從 2 變 3
...
a(n-2) index 從 n-2 變 n-1
```

這些元素的貢獻總共多了：

```text
a0 + a1 + a2 + ... + a(n-2)
```

但是最後一個元素：

```text
a(n-1)
```

原本在 `F(0)` 裡的貢獻是：

```text
(n-1) * a(n-1)
```

旋轉後它跑到 index 0，貢獻變成：

```text
0 * a(n-1) = 0
```

所以它少了：

```text
(n-1) * a(n-1)
```

---

## 9. 遞推公式

最後可以整理出：

```text
F(k) = F(k-1) + sum - n * nums[n-k]
```

這是本題最重要的公式。

---

## 10. 為什麼是 nums[n-k]？

當我們從 `F(k-1)` 變成 `F(k)` 時，代表又向右旋轉一次。

這次被移到最前面的元素是：

```text
nums[n-k]
```

例如：

```text
nums = [4, 3, 2, 6]
n = 4
```

---

### 從 F(0) 到 F(1)

被移到最前面的是：

```text
nums[4-1] = nums[3] = 6
```

所以：

```text
F(1) = F(0) + sum - n * nums[3]
     = 25 + 15 - 4 * 6
     = 25 + 15 - 24
     = 16
```

---

### 從 F(1) 到 F(2)

被移到最前面的是：

```text
nums[4-2] = nums[2] = 2
```

所以：

```text
F(2) = F(1) + sum - n * nums[2]
     = 16 + 15 - 4 * 2
     = 23
```

---

### 從 F(2) 到 F(3)

被移到最前面的是：

```text
nums[4-3] = nums[1] = 3
```

所以：

```text
F(3) = F(2) + sum - n * nums[1]
     = 23 + 15 - 4 * 3
     = 26
```

最大值就是：

```text
26
```

---

## 11. 解題流程

整體流程如下：

```text
1. 計算陣列總和 sum
2. 計算 F(0)
3. 用公式依序算出 F(1) 到 F(n-1)
4. 每次更新最大值
5. 回傳最大值
```

---

## 12. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    int maxRotateFunction(vector<int>& nums) {
        int n = nums.size();

        long long sum = 0;
        long long f = 0;

        for (int i = 0; i < n; i++) {
            sum += nums[i];
            f += (long long)i * nums[i];
        }

        long long ans = f;

        for (int k = 1; k < n; k++) {
            f = f + sum - (long long)n * nums[n - k];
            ans = max(ans, f);
        }

        return (int)ans;
    }
};
```

---

## 13. 程式逐行說明

### 取得陣列長度

```cpp
int n = nums.size();
```

`n` 是陣列長度。

---

### 建立 sum 和 f

```cpp
long long sum = 0;
long long f = 0;
```

`sum` 用來存整個陣列總和。

`f` 用來存目前的旋轉函數值，初始時用來計算 `F(0)`。

雖然題目說答案 fits in 32-bit integer，但中間計算時使用 `long long` 會更安全。

---

### 計算 sum 和 F(0)

```cpp
for (int i = 0; i < n; i++) {
    sum += nums[i];
    f += (long long)i * nums[i];
}
```

這段同時做兩件事：

```text
1. sum += nums[i]
2. f += i * nums[i]
```

最後：

```text
sum = 所有 nums 的總和
f = F(0)
```

---

### 初始化答案

```cpp
long long ans = f;
```

一開始最大值先設成 `F(0)`。

---

### 計算 F(1) 到 F(n-1)

```cpp
for (int k = 1; k < n; k++) {
```

從 `k = 1` 開始，因為 `F(0)` 已經算過了。

---

### 使用遞推公式

```cpp
f = f + sum - (long long)n * nums[n - k];
```

這行是本題核心：

```text
F(k) = F(k-1) + sum - n * nums[n-k]
```

每次用前一次的 `f` 推出新的 `f`。

---

### 更新最大值

```cpp
ans = max(ans, f);
```

每算出一個新的 `F(k)`，就和目前最大值比較。

---

### 回傳答案

```cpp
return (int)ans;
```

題目保證答案 fits in 32-bit integer，所以最後可以轉回 `int`。

---

## 14. 為什麼這樣是 O(n)？

我們做了兩個 loop：

第一個 loop：

```text
計算 sum 和 F(0)
```

花：

```text
O(n)
```

第二個 loop：

```text
計算 F(1) 到 F(n-1)
```

也花：

```text
O(n)
```

所以總時間是：

```text
O(n)
```

## 15. 複雜度分析

### Time Complexity

```text
O(n)
```

只需要掃描陣列兩次。

---

### Space Complexity

```text
O(1)
```

只使用了：

```text
sum
f
ans
n
```

沒有額外建立新陣列。

---

## 16. 為什麼不需要真的 rotate？

因為真的 rotate 會讓每次操作都變成 `O(n)`。

但這題的旋轉之間有規律。

只要知道：

```text
F(k-1)
sum
被移到最前面的元素 nums[n-k]
```

就可以直接算出：

```text
F(k)
```

所以不需要真的建立：

```text
arr0, arr1, arr2, ...
```

---

## 17. 面試講法

### 中文版

這題我不會真的旋轉陣列，因為那樣會造成 `O(n^2)` 的時間複雜度。  
我會先計算整個陣列總和 `sum` 和初始值 `F(0)`。  
接著利用遞推公式 `F(k) = F(k-1) + sum - n * nums[n-k]`，用 `O(1)` 的時間從前一個旋轉函數值推到下一個。  
因此整體只需要 `O(n)` 時間和 `O(1)` 額外空間。

---

### English Version

I avoid rotating the array explicitly because that would lead to `O(n^2)` time complexity.  
Instead, I first compute the total sum of the array and the initial value `F(0)`.  
Then I use the recurrence relation `F(k) = F(k-1) + sum - n * nums[n-k]` to compute each rotation function in constant time.  
This gives an overall time complexity of `O(n)` and space complexity of `O(1)`.

---

## 18. 一句話總結

```text
這題的關鍵是不要真的旋轉陣列，而是利用 F(k) 和 F(k-1) 的關係，用遞推公式在 O(n) 時間內找出最大值。
```
