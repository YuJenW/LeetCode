# 151. Reverse Words in a String 解題說明

## 1. 題目說明

給定一個字串：

```text
s
```

請將字串中的**單字順序反轉**，並回傳新的字串。

---

## 2. 單字的定義

題目中：

```text
A word is defined as a sequence of non-space characters.
```

意思是：

> 一個單字就是一段連續的非空白字元。

例如：

```text
"the sky is blue"
```

可以拆成：

```text
the
sky
is
blue
```

---

## 3. 題目要求

輸出時必須符合：

```text
1. 單字順序反轉
2. 單字之間只保留一個空格
3. 不可以有開頭空格
4. 不可以有結尾空格
5. 原字串中多餘的空格要被移除
```

---

## 4. 題目範例

### Example 1

```text
Input:
s = "the sky is blue"

Output:
"blue is sky the"
```

原本單字順序：

```text
the -> sky -> is -> blue
```

反轉後：

```text
blue -> is -> sky -> the
```

---

### Example 2

```text
Input:
s = "  hello world  "

Output:
"world hello"
```

原字串有：

```text
前導空格
結尾空格
```

輸出時要全部移除。

---

### Example 3

```text
Input:
s = "a good   example"

Output:
"example good a"
```

原本：

```text
good 和 example 中間有多個空格
```

輸出時要變成：

```text
只有一個空格
```

---

## 5. 能應用的場景

這題本質是：

```text
字串分割 + 重組
```

常見應用：

```text
1. 文字前處理
2. 清理多餘空格
3. 句子反轉
4. NLP 輸入格式標準化
5. 指令列文字解析
```

## 6. 需要的知識點

這題主要需要理解：

```text
1. string 字串操作
2. 空白字元判斷
3. 從右往左掃描字串
4. 擷取單字
5. 控制輸出空格數量
6. 避免多餘空格
```

---

# 7. 解法想法

這題最直覺的思路是：

```text
從字串尾端往前掃描
每找到一個完整單字，就加入答案
```

---

## 為什麼從右往左？

因為題目要：

```text
反轉單字順序
```

原本：

```text
the sky is blue
```

如果從右往左找，會依序遇到：

```text
blue
is
sky
the
```

剛好就是答案需要的順序。

---

# 8. 解題流程

我們使用：

```text
i
```

從字串尾端開始往前移動。

每一輪做：

```text
1. 跳過多餘空格
2. 找到目前單字的結尾
3. 繼續往前找到單字開頭
4. 擷取這個單字
5. 加到答案中
6. 若答案不是空的，先補一個空格
```

---

# 9. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    string reverseWords(string s) {
        string ans;
        int i = s.size() - 1;

        while (i >= 0) {
            // 1. 跳過空格
            while (i >= 0 && s[i] == ' ') {
                i--;
            }

            // 如果已經沒有字元，結束
            if (i < 0) {
                break;
            }

            // 2. 記錄單字結尾
            int end = i;

            // 3. 找到單字開頭
            while (i >= 0 && s[i] != ' ') {
                i--;
            }

            // 4. 如果 ans 不是空的，先加一個空格
            if (!ans.empty()) {
                ans += ' ';
            }

            // 5. 加入目前單字
            ans += s.substr(i + 1, end - i);
        }

        return ans;
    }
};
```

# 10. 程式逐行說明

## 10.1 建立答案字串

```cpp
string ans;
```

用來存最後反轉後的結果。

---

## 10.2 從字串尾端開始

```cpp
int i = s.size() - 1;
```

例如：

```text
s = "the sky is blue"
```

一開始：

```text
i 指向 'e'
```

---

## 10.3 跳過尾端空格

```cpp
while (i >= 0 && s[i] == ' ') {
    i--;
}
```

這段的作用是：

```text
跳過結尾空格
也跳過單字之間多餘的連續空格
```

---

### 範例

```text
s = "  hello world  "
```

從最後往前掃時，會先跳過最後兩個空格。

---

## 10.4 如果沒有字元了，就結束

```cpp
if (i < 0) {
    break;
}
```

代表整個字串已經掃描完。

---

## 10.5 記錄這個單字的結尾

```cpp
int end = i;
```

例如：

```text
s = "the sky is blue"
```

當 `i` 指到最後的 `'e'` 時：

```text
end = i
```

這就是單字 `"blue"` 的結尾位置。

---

## 10.6 找出單字開頭

```cpp
while (i >= 0 && s[i] != ' ') {
    i--;
}
```

這段會一路往左走，直到：

```text
遇到空格
或
字串開頭前一格
```

---

### 範例

```text
blue
```

如果：

```text
end 指向 e
```

那麼 while 結束後：

```text
i 會停在 b 前面的空格位置
```

---

## 10.7 為什麼用 `i + 1`？

因為 while 結束時：

```text
i 停在單字前面的空格
```

所以單字真正開始的位置是：

```cpp
i + 1
```

---

## 10.8 為什麼 substring 長度是 `end - i`？

```cpp
s.substr(i + 1, end - i)
```

假設：

```text
i 停在空格位置 10
end 在單字結尾位置 14
```

那單字長度是：

```text
14 - 10 = 4
```

剛好就是單字長度。


# 12. 為什麼要先判斷 `ans.empty()`？

```cpp
if (!ans.empty()) {
    ans += ' ';
}
```

因為答案中的單字之間只需要一個空格。

---

## 第一個單字

答案原本是空的：

```text
ans = ""
```

不需要先加空格。

---

## 後續單字

若答案已有內容，例如：

```text
ans = "blue"
```

接下來加 `"is"` 時，就要先補：

```text
"blue "
```

再加：

```text
"is"
```

變成：

```text
"blue is"
```

# 14. 複雜度分析

假設：

```text
n = s.length()
```

### 時間複雜度

```text
O(n)
```

每個字元最多被掃描一次。

---

### 空間複雜度

```text
O(n)
```

因為需要建立輸出字串 `ans`。

# 15. 另一種解法：stringstream

也可以先把所有單字讀出來，再反轉。

```cpp
class Solution {
public:
    string reverseWords(string s) {
        stringstream ss(s);
        vector<string> words;
        string word;

        while (ss >> word) {
            words.push_back(word);
        }

        string ans;

        for (int i = words.size() - 1; i >= 0; i--) {
            if (!ans.empty()) {
                ans += ' ';
            }
            ans += words[i];
        }

        return ans;
    }
};
```

---

## stringstream 版本優缺點

### 優點

```text
1. 很好懂
2. 自動處理多餘空格
3. 程式較直觀
```

### 缺點

```text
1. 額外使用 vector<string>
2. 空間較多
3. 效率通常不如直接掃描
```

# 16. 哪個版本比較推薦？

| 版本 | 優點 | 缺點 |
|---|---|---|
| 從右往左掃描 | 效率較好，空間較省 | 需要理解 index 操作 |
| stringstream | 最好懂 | 額外空間較多 |

# 17. 面試講法

### 中文版

我從字串尾端往前掃描。  
每次先跳過空格，再找出一個完整單字的範圍，接著把這個單字加入答案字串。  
在加入新單字前，如果答案不為空，就先補一個空格，這樣可以確保輸出格式正確。  
由於每個字元最多掃描一次，因此時間複雜度是 `O(n)`。

---

### English Version

I scan the input string from right to left.  
For each step, I skip extra spaces, locate one complete word, and append it to the result string.  
Before appending a new word, I add a single space only if the result is not empty, which ensures proper formatting.  
Since each character is processed at most once, the time complexity is `O(n)`.
