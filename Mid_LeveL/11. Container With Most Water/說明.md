# 11. Container With Most Water 解題說明

## 1. 題目說明

給定一個整數陣列：

```text
height
```

其中：

```text
height[i]
```

代表在 x 軸位置 `i` 上，有一條高度為 `height[i]` 的垂直線。

我們要選出其中兩條線，搭配 x 軸形成一個容器，並找出：

```text
這個容器最多可以裝多少水
```

---

## 2. 面積怎麼計算？

假設選擇：

```text
left  = 左邊的線
right = 右邊的線
```

容器可以裝的水量是：

```text
寬度 × 高度
```

### 寬度

```text
right - left
```

### 高度

容器的水位會受到較短的那條線限制，所以高度是：

```text
min(height[left], height[right])
```

---

因此面積公式是：

```text
area = (right - left) * min(height[left], height[right])
```

---

## 3. 題目範例

### Example 1

```text
height = [1,8,6,2,5,4,8,3,7]
```

最大面積由：

```text
height[1] = 8
height[8] = 7
```

形成。

寬度：

```text
8 - 1 = 7
```

高度：

```text
min(8, 7) = 7
```

面積：

```text
7 × 7 = 49
```

所以答案是：

```text
49
```

---

## 4. 能應用的場景

這題屬於典型的：

```text
Two Pointers 雙指標最佳化問題
```

類似的應用包含：

```text
1. 在有左右邊界的情況下找最大面積
2. 區間最佳化問題
3. 透過移動指標縮小搜尋範圍
4. 用條件判斷排除不可能更好的解
```

這題真正重要的不是面積公式，而是：

```text
如何從 O(n²) 優化到 O(n)
```

## 6. 暴力解法

最直覺的做法是：

```text
枚舉所有兩條線的組合
```

例如：

```cpp
for each left
    for each right
        計算面積
```

---

### 暴力法程式概念

```cpp
for (int left = 0; left < n; left++) {
    for (int right = left + 1; right < n; right++) {
        int area = (right - left) * min(height[left], height[right]);
        ans = max(ans, area);
    }
}
```

---

### 暴力法複雜度

```text
Time Complexity: O(n²)
Space Complexity: O(1)
```

---

### 為什麼不夠好？

題目限制：

```text
2 <= n <= 100000
```

如果 `n = 100000`：

```text
O(n²) = 10^10
```

會超時。

所以必須設計更快的解法。

---

## 7. 最佳解法：雙指標

我們使用兩個指標：

```text
left  指向陣列最左邊
right 指向陣列最右邊
```

初始：

```text
left = 0
right = n - 1
```

這樣一開始容器的寬度最大。

---

## 8. 雙指標核心流程

每一輪：

```text
1. 計算目前 left 和 right 形成的面積
2. 更新最大面積
3. 移動較短的那一邊
```

---

## 9. 為什麼要移動較短的那一邊？

這是這題最關鍵的地方。

假設目前：

```text
height[left] < height[right]
```

那麼當前容器高度由：

```text
height[left]
```

決定。

因為水位不能超過較短的牆。

---

### 如果移動右邊會怎樣？

寬度會變小：

```text
right - left 變小
```

但高度仍然最多只能被：

```text
height[left]
```

限制。

因為左邊那條短牆沒有變。

所以面積不可能變大。

---

### 因此，應該移動左邊

只有移動較短的左邊，才有機會找到：

```text
更高的左邊界
```

進而可能讓面積變大。

---

## 10. 用圖像直覺理解

假設：

```text
left  高度 = 3
right 高度 = 8
```

面積高度只能用：

```text
min(3, 8) = 3
```

如果你移動右邊：

```text
right 可能變成 7 或 10
```

但左邊仍然是 3。

所以高度仍然是：

```text
3
```

同時寬度還縮小了。

因此沒有變大的希望。

## 11. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int left = 0;
        int right = height.size() - 1;
        int ans = 0;

        while (left < right) {
            int width = right - left;
            int h = min(height[left], height[right]);
            int area = width * h;

            ans = max(ans, area);

            if (height[left] < height[right]) {
                left++;
            } else {
                right--;
            }
        }

        return ans;
    }
};
```

## 13. 程式逐行說明

### 13.1 建立左右指標

```cpp
int left = 0;
int right = height.size() - 1;
```

一開始選最左邊和最右邊的線。

這樣寬度是最大的。

---

### 13.2 建立答案變數

```cpp
int ans = 0;
```

用來記錄目前找到的最大面積。

---

### 13.3 當左右指標還沒有相遇

```cpp
while (left < right) {
```

只要還有兩條不同的線，就繼續計算。

---

### 13.4 計算寬度

```cpp
int width = right - left;
```

例如：

```text
left = 1
right = 8
```

則：

```text
width = 8 - 1 = 7
```

---

### 13.5 計算高度

```cpp
int h = min(height[left], height[right]);
```

容器高度由較短的牆決定。

---

### 13.6 計算面積

```cpp
int area = width * h;
```

---

### 13.7 更新最大值

```cpp
ans = max(ans, area);
```

如果目前面積更大，就更新答案。

---

### 13.8 移動較短的那一邊

```cpp
if (height[left] < height[right]) {
    left++;
} else {
    right--;
}
```

如果左邊較短：

```text
移動 left
```

如果右邊較短或一樣高：

```text
移動 right
```

## 14. 範例完整走一次

### 輸入

```text
height = [1,8,6,2,5,4,8,3,7]
```

---

### 第 1 輪

```text
left = 0, height[left] = 1
right = 8, height[right] = 7
```

面積：

```text
width = 8
height = min(1, 7) = 1
area = 8 × 1 = 8
```

目前：

```text
ans = 8
```

因為左邊較短：

```text
left++
```

---

### 第 2 輪

```text
left = 1, height[left] = 8
right = 8, height[right] = 7
```

面積：

```text
width = 7
height = min(8, 7) = 7
area = 7 × 7 = 49
```

更新：

```text
ans = 49
```

因為右邊較短：

```text
right--
```

---

### 後續

雖然後面仍會繼續檢查其他組合，但沒有任何組合超過：

```text
49
```

所以最終答案為：

```text
49
```

## 15. 為什麼這個方法不會漏掉正確答案？

因為每次移動較短邊時，我們只排除了：

```text
不可能產生更大面積的組合
```

假設左邊較短：

```text
height[left] <= height[right]
```

如果保留左邊、只移動右邊：

```text
1. 寬度變小
2. 高度上限仍受 height[left] 限制
```

因此面積一定不會更大。

所以移動較短邊是安全的。

## 16. 複雜度分析

假設：

```text
n = height.length
```

### 時間複雜度

```text
O(n)
```

因為：

```text
left 最多往右移 n 次
right 最多往左移 n 次
```

整體只會掃描一次。

---

### 空間複雜度

```text
O(1)
```

只使用固定幾個變數：

```text
left
right
width
h
area
ans
```

## 18. 面試講法

### 中文版

我使用雙指標解這題。  
一開始將 `left` 放在最左邊，`right` 放在最右邊，這樣可以先取得最大寬度。  
每一輪計算目前容器面積：

```text
(right - left) × min(height[left], height[right])
```

接著移動較短的那一邊，因為目前容器高度由較短邊決定。如果移動較高邊，寬度變小但高度上限不會提升，因此不可能得到更大面積。  
這樣可以在 `O(n)` 時間內找到最大值。

---

### English Version

I use a two-pointer approach.  
Initially, I place one pointer at the beginning and the other at the end to maximize the width.  
At each step, I compute the current area as:

```text
(right - left) * min(height[left], height[right])
```

Then I move the pointer at the shorter line, because the shorter line limits the height of the container.  
Moving the taller line would only reduce the width without improving the limiting height.  
This gives an `O(n)` time solution with `O(1)` extra space.

---

## 19. 一句話總結

```text
這題的核心是：面積由「寬度 × 較短邊高度」決定，因此每次只移動較短的那一邊，才能有機會找到更大的答案。
```
