# 4. Median of Two Sorted Arrays 解題說明

## 1. 題目說明

給定兩個已經排序好的陣列：

```text
nums1
nums2
```

它們的長度分別是：

```text
m
n
```

請找出這兩個陣列合併後的中位數 median。

---

## 2. 什麼是中位數？

中位數就是：

```text
排序後，位在中間的數字
```

---

### 總長度是奇數

例如：

```text
[1, 2, 3]
```

中間是：

```text
2
```

所以 median 是：

```text
2
```

---

### 總長度是偶數

例如：

```text
[1, 2, 3, 4]
```

中間有兩個數：

```text
2 和 3
```

所以 median 是：

```text
(2 + 3) / 2 = 2.5
```

---

## 3. 題目範例

### Example 1

```text
nums1 = [1,3]
nums2 = [2]
```

合併後：

```text
[1,2,3]
```

總長度是 `3`，是奇數。

中位數是：

```text
2
```

所以輸出：

```text
2.00000
```

---

### Example 2

```text
nums1 = [1,2]
nums2 = [3,4]
```

合併後：

```text
[1,2,3,4]
```

總長度是 `4`，是偶數。

中間兩個數是：

```text
2 和 3
```

所以：

```text
median = (2 + 3) / 2 = 2.5
```

輸出：

```text
2.50000
```

---

## 4. 題目要求的時間複雜度

題目要求：

```text
O(log(m+n))
```

這代表不能單純合併兩個陣列後再找中位數。

因為完整合併需要：

```text
O(m+n)
```

但這題希望更快。


# 第一種解法：合併陣列

## 6. 直覺解法

最直覺的方式是：

```text
1. 把 nums1 和 nums2 合併成一個 sorted array
2. 根據總長度判斷中位數
```

---

## 7. 合併法程式碼

```cpp
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        vector<int> merged;

        int i = 0;
        int j = 0;

        while (i < nums1.size() && j < nums2.size()) {
            if (nums1[i] < nums2[j]) {
                merged.push_back(nums1[i]);
                i++;
            } else {
                merged.push_back(nums2[j]);
                j++;
            }
        }

        while (i < nums1.size()) {
            merged.push_back(nums1[i]);
            i++;
        }

        while (j < nums2.size()) {
            merged.push_back(nums2[j]);
            j++;
        }

        int total = merged.size();

        if (total % 2 == 1) {
            return merged[total / 2];
        }

        return (merged[total / 2 - 1] + merged[total / 2]) / 2.0;
    }
};
```

---

## 8. 合併法的問題

這個方法很好懂，但時間複雜度是：

```text
O(m+n)
```

空間複雜度是：

```text
O(m+n)
```

不符合題目要求的：

```text
O(log(m+n))
```

所以這題真正要練的是：

```text
Binary Search + Partition
```

# 第二種解法：Binary Search + Partition

## 9. 核心想法

不要真的合併陣列。

我們要做的是：

```text
把 nums1 和 nums2 切成左半邊與右半邊
```

讓：

```text
左半邊的數量 = 右半邊的數量
或
左半邊比右半邊多 1 個
```

並且滿足：

```text
左半邊所有數 <= 右半邊所有數
```

只要找到這個切法，中位數就出來了。

---

## 10. 用圖理解 partition

假設：

```text
nums1 = [1, 3]
nums2 = [2]
```

合併後應該是：

```text
[1, 2, 3]
```

我們希望切成：

```text
左半邊：[1, 2]
右半邊：[3]
```

因為總長度是奇數，所以左半邊可以多一個。

中位數就是左半邊最大值：

```text
2
```

---

## 11. 一般化切割方式

假設：

```text
nums1 被切成：
nums1 左半邊 | nums1 右半邊

nums2 被切成：
nums2 左半邊 | nums2 右半邊
```

我們要讓：

```text
nums1 左半邊數量 + nums2 左半邊數量
=
(total + 1) / 2
```

---

## 12. 為什麼是 `(total + 1) / 2`？

令：

```text
total = m + n
```

---

### 如果 total 是奇數

例如：

```text
total = 5
```

那：

```text
(total + 1) / 2 = 3
```

左邊有 3 個，右邊有 2 個。

中位數是：

```text
左半邊最大值
```

---

### 如果 total 是偶數

例如：

```text
total = 4
```

那：

```text
(total + 1) / 2 = 2
```

左邊 2 個，右邊 2 個。

中位數是：

```text
(左半邊最大值 + 右半邊最小值) / 2
```

---

## 13. 四個關鍵值

假設我們切在：

```text
nums1 的 cut1
nums2 的 cut2
```

我們只需要關心切口附近的四個值：

```text
left1  = nums1 左半邊最大值
right1 = nums1 右半邊最小值

left2  = nums2 左半邊最大值
right2 = nums2 右半邊最小值
```

---

圖像如下：

```text
nums1:  ... left1 | right1 ...
nums2:  ... left2 | right2 ...
```

---

## 14. 正確切割條件

如果切割正確，必須滿足：

```text
left1 <= right2
left2 <= right1
```

為什麼？

因為我們要確保：

```text
左半邊所有數 <= 右半邊所有數
```

只需要檢查交界處最大與最小即可。

---

## 15. 找到正確切割後怎麼算 median？

### 如果總長度是奇數

```cpp
median = max(left1, left2);
```

因為左半邊多一個，中位數就是左半邊最大值。

---

### 如果總長度是偶數

```cpp
median = (max(left1, left2) + min(right1, right2)) / 2.0;
```

因為中位數是中間兩個數的平均。

---

## 16. 為什麼對比較短的陣列做 binary search？

我們只對：

```text
較短的陣列
```

做 binary search。

原因：

```text
1. 搜尋範圍比較小
2. 比較容易處理邊界
3. 可以保證 cut2 不會超出 nums2 的範圍
```

所以程式一開始會寫：

```cpp
if (nums1.size() > nums2.size()) {
    return findMedianSortedArrays(nums2, nums1);
}
```

確保：

```text
nums1 是比較短的陣列
```

---

## 17. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        if (nums1.size() > nums2.size()) {
            return findMedianSortedArrays(nums2, nums1);
        }

        int m = nums1.size();
        int n = nums2.size();

        int left = 0;
        int right = m;
        int totalLeft = (m + n + 1) / 2;

        while (left <= right) {
            int cut1 = left + (right - left) / 2;
            int cut2 = totalLeft - cut1;

            int left1 = (cut1 == 0) ? INT_MIN : nums1[cut1 - 1];
            int right1 = (cut1 == m) ? INT_MAX : nums1[cut1];

            int left2 = (cut2 == 0) ? INT_MIN : nums2[cut2 - 1];
            int right2 = (cut2 == n) ? INT_MAX : nums2[cut2];

            if (left1 <= right2 && left2 <= right1) {
                if ((m + n) % 2 == 1) {
                    return max(left1, left2);
                }

                return (max(left1, left2) + min(right1, right2)) / 2.0;
            } 
            else if (left1 > right2) {
                right = cut1 - 1;
            } 
            else {
                left = cut1 + 1;
            }
        }

        return 0.0;
    }
};
```

## 19. 程式逐行說明

### 19.1 確保 nums1 比較短

```cpp
if (nums1.size() > nums2.size()) {
    return findMedianSortedArrays(nums2, nums1);
}
```

如果 `nums1` 比 `nums2` 長，就交換兩者。

這樣 binary search 會發生在較短的陣列上。

---

### 19.2 取得長度

```cpp
int m = nums1.size();
int n = nums2.size();
```

---

### 19.3 設定 binary search 範圍

```cpp
int left = 0;
int right = m;
```

`cut1` 可以從：

```text
0 到 m
```

代表：

```text
nums1 左半邊可以拿 0 個到 m 個元素
```

---

### 19.4 計算左半邊應該有幾個元素

```cpp
int totalLeft = (m + n + 1) / 2;
```

這代表：

```text
合併後左半邊應該有多少元素
```

---

### 19.5 計算 cut1 和 cut2

```cpp
int cut1 = left + (right - left) / 2;
int cut2 = totalLeft - cut1;
```

`cut1` 是 `nums1` 左半邊拿幾個。

`cut2` 是 `nums2` 左半邊拿幾個。

兩者加起來必須等於：

```text
totalLeft
```

---

## 20. `cut1` 是什麼？

如果：

```text
nums1 = [1, 3, 8]
cut1 = 2
```

意思是：

```text
nums1 左半邊拿 2 個元素
```

切法：

```text
[1, 3] | [8]
```

---

## 21. 邊界值處理

### 如果 cut1 == 0

代表：

```text
nums1 左半邊沒有元素
```

那就沒有 `left1`。

所以設定：

```cpp
left1 = INT_MIN;
```

意思是：

```text
讓它小到不影響比較
```

---

### 如果 cut1 == m

代表：

```text
nums1 右半邊沒有元素
```

那就沒有 `right1`。

所以設定：

```cpp
right1 = INT_MAX;
```

意思是：

```text
讓它大到不影響比較
```

---

### 對 nums2 也是同理

```cpp
int left2 = (cut2 == 0) ? INT_MIN : nums2[cut2 - 1];
int right2 = (cut2 == n) ? INT_MAX : nums2[cut2];
```

---

## 22. 為什麼要用 INT_MIN / INT_MAX？

它們是哨兵值 sentinel。

目的是避免越界。

例如：

```text
nums1 左半邊沒有元素
```

我們不能寫：

```cpp
nums1[cut1 - 1]
```

因為會變成：

```cpp
nums1[-1]
```

這是錯的。

所以用：

```cpp
INT_MIN
```

代表「左邊沒有值」。

---

## 23. 判斷切割是否正確

```cpp
if (left1 <= right2 && left2 <= right1)
```

如果成立，代表：

```text
左半邊所有數都 <= 右半邊所有數
```

切割成功。

---

## 24. 如果切割成功，怎麼回傳？

### 總長度是奇數

```cpp
if ((m + n) % 2 == 1) {
    return max(left1, left2);
}
```

左半邊多一個元素，中位數就是左半邊最大值。

---

### 總長度是偶數

```cpp
return (max(left1, left2) + min(right1, right2)) / 2.0;
```

中位數是中間兩個數的平均。

---

## 25. 如果 `left1 > right2` 怎麼辦？

```cpp
else if (left1 > right2) {
    right = cut1 - 1;
}
```

代表：

```text
nums1 左半邊拿太多了
```

因為 `left1` 太大，竟然比 `right2` 還大。

所以要讓 `cut1` 往左移。

---

## 26. 如果 `left2 > right1` 怎麼辦？

```cpp
else {
    left = cut1 + 1;
}
```

代表：

```text
nums1 左半邊拿太少了
```

因為 `nums2` 左半邊的最大值 `left2` 太大，放到左邊會破壞順序。

所以要讓 `cut1` 往右移，從 `nums1` 多拿一些元素到左半邊。

```text
nums1 = [1,3]
nums2 = [2]
```

```text
nums1 = [2]
```

### 初始

```text
m = 1
n = 2
totalLeft = (1 + 2 + 1) / 2 = 2

left = 0
right = 1
```

### 第 1 次 binary search

```text
cut1 = 0
cut2 = 2
```

切法：

```text
nums1: [] | [2]
nums2: [1,3] | []
```

四個值：

```text
left1  = INT_MIN
right1 = 2

right2 = INT_MAX
```

檢查：

```text
left1 <= right2  成立
left2 <= right1  不成立，因為 3 > 2
```

```text
left = cut1 + 1 = 1
```

```text
cut1 = 1
cut2 = 1
```

切法：

```text
nums1: [2] | []
nums2: [1] | [3]
```

四個值：

```text
left1  = 2
right1 = INT_MAX
left2  = 1
right2 = 3
```

檢查：

```text
left1 <= right2  2 <= 3 成立
left2 <= right1  1 <= INT_MAX 成立
```

切割正確。


## 31. 面試講法

### 中文版

這題我使用 binary search 搭配 partition 的概念。  
我不真正合併兩個陣列，而是嘗試把兩個陣列切成左右兩半，使得左半邊元素數量等於 `(m+n+1)/2`，並且左半邊所有元素都小於等於右半邊所有元素。  
我只在較短的陣列上二分搜尋切割位置，另一個陣列的切割位置可以由左半邊總數推得。  
當 `left1 <= right2` 且 `left2 <= right1` 時，代表切割正確。  
若總長度為奇數，中位數是左半邊最大值；若為偶數，中位數是左半邊最大值與右半邊最小值的平均。

---

### English Version

I use binary search with the partition idea.  
Instead of merging the arrays, I try to partition both arrays into left and right halves such that the left half contains `(m+n+1)/2` elements and every element in the left half is less than or equal to every element in the right half.  
I perform binary search on the shorter array, and the partition position in the other array can be derived from the required left-half size.  
When `left1 <= right2` and `left2 <= right1`, the partition is valid.  
If the total length is odd, the median is the maximum value on the left side; otherwise, it is the average of the maximum value on the left side and the minimum value on the right side.

## 32. 一句話總結

```text
這題的核心是不用真的合併兩個陣列，而是用二分搜尋找到正確切割點，讓左半邊與右半邊剛好形成中位數分界。

------------------------------------------------------------------------------------------

Binary Search 的目標是找 nums1 的切割位置 cut1。

先算出合併後左半邊應該有幾個元素 totalLeft。
接著猜 nums1 左邊拿 cut1 個，nums2 左邊就拿 cut2 = totalLeft - cut1 個。

切完後，只要確認：
left1 <= right2
left2 <= right1

就代表左半邊所有元素都小於等於右半邊，切割正確。

如果 left1 > right2，代表 nums1 左邊拿太多，cut1 要往左。
如果 left2 > right1，代表 nums1 左邊拿太少，cut1 要往右。

```
