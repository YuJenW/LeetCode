# 2215. Find the Difference of Two Arrays 解題說明

## 1. 題目說明

給定兩個整數陣列：

```text
nums1
nums2
```

請回傳一個大小為 `2` 的二維陣列：

```text
answer[0]
answer[1]
```

其中：

```text
answer[0]：
nums1 中有出現，但 nums2 中沒有出現的「不重複」數字

answer[1]：
nums2 中有出現，但 nums1 中沒有出現的「不重複」數字
```

---

## 2. 題目重點

這題要注意兩件事：

### 2.1 要找兩個陣列彼此獨有的數字

例如：

```text
nums1 = [1,2,3]
nums2 = [2,4,6]
```

分析：

```text
1 只在 nums1
2 兩邊都有
3 只在 nums1
4 只在 nums2
6 只在 nums2
```

所以答案是：

```text
[[1,3],[4,6]]
```

---

### 2.2 答案不能重複

例如：

```text
nums1 = [1,2,3,3]
nums2 = [1,1,2,2]
```

雖然 `3` 在 `nums1` 出現兩次，但答案只需要放一次：

```text
[[3],[]]
```

---

## 3. 題目範例

### Example 1

```text
Input:
nums1 = [1,2,3]
nums2 = [2,4,6]

Output:
[[1,3],[4,6]]
```

---

### Example 2

```text
Input:
nums1 = [1,2,3,3]
nums2 = [1,1,2,2]

Output:
[[3],[]]
```

---

## 4. 這題的核心需求

這題其實需要完成三件事：

```text
1. 紀錄 nums1 出現過哪些數字
2. 紀錄 nums2 出現過哪些數字
3. 找出只存在其中一邊的數字
```

另外還要確保：

```text
答案不重複
```

# 第一種解法：利用數值範圍，使用 bool[2001]

## 5. 為什麼可以用 bool[2001]？

題目限制：

```text
-1000 <= nums1[i], nums2[i] <= 1000
```

代表所有可能出現的數字只有：

```text
-1000, -999, ..., 0, ..., 999, 1000
```

總共有：

```text
2001 種可能值
```

因此我們可以準備兩個布林陣列：

```cpp
bool inNums1[2001] = {};
bool inNums2[2001] = {};
```

分別記錄：

```text
某個數字是否出現在 nums1
某個數字是否出現在 nums2
```

---

## 6. 為什麼需要加 1000？

陣列 index 不能是負數。

但題目中的數字可能是：

```text
-1000
```

所以我們用：

```text
index = value + 1000
```

把所有值平移到合法陣列索引。

---

### 對應關係

| 原始數字 value | 對應 index = value + 1000 |
|---:|---:|
| -1000 | 0 |
| -999 | 1 |
| -1 | 999 |
| 0 | 1000 |
| 1 | 1001 |
| 999 | 1999 |
| 1000 | 2000 |

---

## 7. bool 陣列如何記錄「有沒有出現」？

假設：

```text
nums1 = [1,2,3,3]
```

我們做：

```cpp
for (int num : nums1) {
    inNums1[num + 1000] = true;
}
```

---

### 過程

```text
1  -> inNums1[1001] = true
2  -> inNums1[1002] = true
3  -> inNums1[1003] = true
3  -> inNums1[1003] = true
```

即使 `3` 出現兩次，最後也只是：

```text
inNums1[1003] = true
```

所以它自然就達成：

```text
去除重複
```

---

## 9. LeetCode 可直接提交程式碼：bool[2001] 版本

```cpp
class Solution {
public:
    vector<vector<int>> findDifference(vector<int>& nums1, vector<int>& nums2) {
        bool inNums1[2001] = {};
        bool inNums2[2001] = {};

        // 記錄 nums1 中出現過的數字
        for (int num : nums1) {
            inNums1[num + 1000] = true;
        }

        // 記錄 nums2 中出現過的數字
        for (int num : nums2) {
            inNums2[num + 1000] = true;
        }

        vector<vector<int>> answer(2);

        // 檢查所有可能值：-1000 ~ 1000
        for (int value = -1000; value <= 1000; value++) {
            int idx = value + 1000;

            if (inNums1[idx] && !inNums2[idx]) {
                answer[0].push_back(value);
            }

            if (inNums2[idx] && !inNums1[idx]) {
                answer[1].push_back(value);
            }
        }

        return answer;
    }
};
```

## 12. bool[2001] 版本為什麼高效？

這個版本不需要 Hash Table。

查詢某個值有沒有出現，只要：

```cpp
inNums1[idx]
inNums2[idx]
```

直接存取陣列即可。

---

### 優點

```text
1. 查詢速度非常直接
2. 不需要 unordered_set
3. 自動去除重複值
4. 記憶體大小固定
5. 題目值域小時，通常會比 hash 結構更快
```

---

### 缺點

```text
1. 依賴題目的數值範圍
2. 如果數值範圍很大，就不適合
```

例如如果題目改成：

```text
-10^9 <= nums[i] <= 10^9
```

就不適合開一個超大陣列來記錄。

# 第二種解法：使用 unordered_set

## 14. unordered_set 是什麼？

`unordered_set` 是 C++ STL 中的：

```text
Hash Set
```

它具有兩個重要特性：

```text
1. 不儲存重複元素
2. 平均可以 O(1) 查找某個值是否存在
```

---

## 15. 為什麼這題也適合 unordered_set？

這題需要：

```text
1. 去除重複值
2. 快速判斷某個值是否存在於另一個陣列
```

而 `unordered_set` 剛好很適合。

---

## 16. 解題邏輯

### Step 1：把 nums1 放進 set1

```cpp
unordered_set<int> set1(nums1.begin(), nums1.end());
```

---

### Step 2：把 nums2 放進 set2

```cpp
unordered_set<int> set2(nums2.begin(), nums2.end());
```

這樣可以自動去除重複。

---

### Step 3：找 set1 有、set2 沒有的數字

```cpp
for (int num : set1) {
    if (set2.find(num) == set2.end()) {
        answer[0].push_back(num);
    }
}
```

---

### Step 4：找 set2 有、set1 沒有的數字

```cpp
for (int num : set2) {
    if (set1.find(num) == set1.end()) {
        answer[1].push_back(num);
    }
}
```

## 17. LeetCode 可直接提交程式碼：unordered_set 版本

```cpp
class Solution {
public:
    vector<vector<int>> findDifference(vector<int>& nums1, vector<int>& nums2) {
        unordered_set<int> set1(nums1.begin(), nums1.end());
        unordered_set<int> set2(nums2.begin(), nums2.end());

        vector<vector<int>> answer(2);

        for (int num : set1) {
            if (set2.find(num) == set2.end()) {
                answer[0].push_back(num);
            }
        }

        for (int num : set2) {
            if (set1.find(num) == set1.end()) {
                answer[1].push_back(num);
            }
        }

        return answer;
    }
};
```

## 22. 面試講法

### 中文版

這題我會先觀察 constraints。  
由於數值範圍固定在 `-1000` 到 `1000`，總共只有 `2001` 種可能值，因此可以使用兩個固定大小的 `bool` 陣列來記錄某個數字是否出現在 `nums1` 或 `nums2` 中。  
接著掃描所有可能值，找出只存在其中一邊的數字即可。  
這樣不需要 `unordered_set`，而且空間是固定的 `O(1)`。

如果題目沒有這麼小的數值範圍，則可以改用 `unordered_set`，利用它自動去重並提供平均 `O(1)` 查找。

---

### English Version

I first inspect the constraints.  
Since every value lies between `-1000` and `1000`, there are only `2001` possible values.  
Therefore, I can use two fixed-size boolean arrays to record whether a value appears in `nums1` or `nums2`.  
Then I iterate through all possible values and collect those that appear in exactly one array.  
This avoids using `unordered_set` and keeps the extra space constant.

If the value range were not small and bounded, I would instead use `unordered_set` for average `O(1)` membership checks and duplicate removal.

## 24. `bool[2001]` 與 `unordered_set` 兩種方法的優缺點比較

這題有兩種常見解法：

```text
方法一：bool[2001] 固定陣列標記
方法二：unordered_set 集合查找
```

兩者都能正確解題，但因為這題的數值範圍很小：

```text
-1000 <= nums1[i], nums2[i] <= 1000
```

所以：

```text
bool[2001] 通常會比 unordered_set 執行時間更快，記憶體也更省。
```

# 方法一：`bool[2001]`

## 優點 1：查找方式非常直接

```cpp
inNums1[num + 1000]
```

這是：

```text
直接用 index 取得結果
```

不需要：

```text
計算 hash
尋找 bucket
處理 collision
```

所以速度通常比 `unordered_set` 更快。

---

## 優點 2：記憶體使用固定且很小

```cpp
bool inNums1[2001] = {};
bool inNums2[2001] = {};
```

總共大約是：

```text
2001 + 2001 個 bool
```

記憶體大小固定，不會因為陣列長度不同而大幅增加。

因此在 LeetCode 上通常會看到：

```text
Memory Usage 較低
```

---

## 優點 3：自動去除重複值

例如：

```text
nums1 = [1,2,3,3]
```

執行：

```cpp
inNums1[1003] = true;
inNums1[1003] = true;
```

最後仍然只是：

```text
true
```

所以天然就完成：

```text
去重複
```

---

## 優點 4：這題特別適合

因為值域只有：

```text
2001 種可能數字
```

掃描整個 `-1000 ~ 1000` 的成本很低。

因此這題用固定陣列是很漂亮的解法。

---

## 缺點：依賴數值範圍

如果題目改成：

```text
-10^9 <= nums[i] <= 10^9
```

就不可能開一個超大的陣列來記錄。

所以：

```text
bool[2001] 是針對這題 constraints 設計的特殊最佳化。
```

---

# 方法二：`unordered_set`

## 優點 1：泛用性高

```cpp
unordered_set<int> set1;
unordered_set<int> set2;
```

不需要知道數值範圍。

不管數字是：

```text
-1000 ~ 1000
```

還是：

```text
-10^9 ~ 10^9
```

都可以使用。

---

## 優點 2：語意自然

`unordered_set` 本身就是：

```text
不重複集合
```

很符合這題需求：

```text
1. 去重複
2. 判斷某數字是否存在
```

程式也相對直觀。

---

## 缺點 1：記憶體成本較高

`unordered_set` 不只是存一個整數而已。

它底層還需要管理：

```text
bucket
node
hash 結構
鏈結資訊或內部結構
```

所以記憶體通常會比：

```cpp
bool[2001]
```

高不少。

---

## 缺點 2：速度通常較慢

查詢：

```cpp
set2.find(num)
```

雖然平均是：

```text
O(1)
```

但實際上還包含：

```text
1. 計算 hash
2. 找 bucket
3. 確認元素是否存在
```

而 `bool[2001]` 是：

```text
一次陣列索引就完成
```

所以在這題中：

```text
bool[2001] 通常 runtime 更好。
```

---

### English Version

I would prefer the `bool[2001]` approach for this problem because the constraints limit all values to the range `[-1000, 1000]`.  
By using `num + 1000` as an index, I can directly mark whether a value appears in `nums1` or `nums2`.  
This gives very fast direct array access without hash computation, so it is usually faster than `unordered_set` for this problem.  
It also uses less memory because the boolean arrays have fixed small size.  

If the value range were large or unknown, I would choose `unordered_set` instead for better generality.
