# 643. Maximum Average Subarray I 解題說明

## 1. 題目說明

給定：

```text
一個整數陣列 nums
一個整數 k
```

請找出：

```text
長度剛好等於 k 的連續子陣列
```

使得這段子陣列的：

```text
平均值 maximum average
```

最大。

最後回傳這個最大平均值。

---

## 2. 關鍵字解釋

### 2.1 Contiguous Subarray 是什麼？

`contiguous subarray` 指的是：

```text
陣列中連續的一段元素
```

例如：

```text
nums = [1, 12, -5, -6, 50, 3]
```

以下是長度為 4 的連續子陣列：

```text
[1, 12, -5, -6]
[12, -5, -6, 50]
[-5, -6, 50, 3]
```

但：

```text
[1, -5, 50, 3]
```

不是 contiguous，因為中間跳過了元素。

---

## 3. 題目範例

### Example 1

```text
Input:
nums = [1,12,-5,-6,50,3]
k = 4
```

所有長度為 4 的連續子陣列：

```text
[1, 12, -5, -6]
sum = 1 + 12 - 5 - 6 = 2
average = 2 / 4 = 0.5
```

```text
[12, -5, -6, 50]
sum = 12 - 5 - 6 + 50 = 51
average = 51 / 4 = 12.75
```

```text
[-5, -6, 50, 3]
sum = -5 - 6 + 50 + 3 = 42
average = 42 / 4 = 10.5
```

最大平均值是：

```text
12.75
```

所以輸出：

```text
12.75000
```

---

### Example 2

```text
Input:
nums = [5]
k = 1
```

唯一的子陣列：

```text
[5]
```

平均值：

```text
5 / 1 = 5
```

所以輸出：

```text
5.00000
```

---

## 4. 這題真正要解決什麼？

題目要找：

```text
所有長度為 k 的連續子陣列中，平均值最大的那一段
```

因為：

```text
平均值 = 總和 / k
```

而這題所有子陣列長度都固定是 `k`。

所以：

```text
要讓平均值最大
等價於
讓子陣列總和最大
```

因此這題其實可以轉換成：

> 找出長度為 `k` 的連續子陣列中，最大總和是多少。

最後再除以 `k`。

---

## 5. 能應用的場景

這題的核心技巧是：

```text
Sliding Window 滑動視窗
```

常見應用場景：

```text
1. 固定長度區間的最大總和
2. 固定長度區間的最大平均值
3. 一段連續時間內的最高平均溫度
4. 一段固定天數內的最高銷售額
5. 固定大小視窗中的訊號強度分析
6. 影像或序列資料中的局部區間統計
```

## 6. 需要的知識點

這題主要需要理解：

```text
1. Array 陣列走訪
2. Contiguous Subarray 連續子陣列
3. Average 平均值
4. Sliding Window 滑動視窗
5. 固定長度視窗如何更新總和
6. 為什麼不用每次重新加總
```

## 7. 暴力解法

最直覺的方式是：

```text
枚舉每一個長度為 k 的子陣列
然後重新計算它的總和
```

---

### 暴力法概念

```cpp
for 每個起點 i:
    sum = 0

    for j 從 i 到 i + k - 1:
        sum += nums[j]

    更新最大 sum
```

---

### 暴力法時間複雜度

假設陣列長度是 `n`。

總共有大約：

```text
n - k + 1
```

個子陣列。

每個子陣列都要花：

```text
O(k)
```

重新加總。

所以總時間複雜度是：

```text
O((n-k+1) × k)
```

最壞情況下接近：

```text
O(nk)
```

當：

```text
n = 100000
```

時，可能會太慢。

---

## 8. 最佳解法：Sliding Window

### 核心想法

我們不需要每次重新計算整個長度為 `k` 的區間總和。

因為相鄰兩個視窗只差：

```text
移除最左邊一個元素
加入最右邊一個新元素
```

---

## 9. 用範例理解滑動視窗

```text
nums = [1,12,-5,-6,50,3]
k = 4
```

---

### 第一個視窗

```text
[1,12,-5,-6]
```

總和：

```text
sum = 1 + 12 - 5 - 6 = 2
```

---

### 第二個視窗

從：

```text
[1,12,-5,-6]
```

滑到：

```text
[12,-5,-6,50]
```

我們不需要重新算：

```text
12 + (-5) + (-6) + 50
```

只要：

```text
原本 sum
- 移除離開視窗的 1
+ 加入新進視窗的 50
```

也就是：

```text
new sum = 2 - 1 + 50 = 51
```

---

### 第三個視窗

從：

```text
[12,-5,-6,50]
```

滑到：

```text
[-5,-6,50,3]
```

更新方式：

```text
new sum = 51 - 12 + 3 = 42
```

---

## 10. Sliding Window 的更新公式

假設：

```text
目前視窗總和 = windowSum
```

當視窗往右移一格：

```text
windowSum = windowSum - 離開視窗的元素 + 新加入的元素
```

在程式中就是：

```cpp
windowSum += nums[right];
windowSum -= nums[right - k];
```

---

## 11. 解題流程

整體流程如下：

```text
1. 先計算前 k 個元素的總和
2. 將這個總和當作目前最大總和 maxSum
3. 接著從第 k 個元素開始往右滑動視窗
4. 每次：
   - 加入新元素
   - 移除舊元素
   - 更新 maxSum
5. 最後回傳 maxSum / k
```

---

## 12. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    double findMaxAverage(vector<int>& nums, int k) {
        int windowSum = 0;

        // 先計算第一個長度為 k 的視窗總和
        for (int i = 0; i < k; i++) {
            windowSum += nums[i];
        }

        int maxSum = windowSum;

        // 滑動視窗
        for (int right = k; right < nums.size(); right++) {
            windowSum += nums[right];       // 加入右邊的新元素
            windowSum -= nums[right - k];   // 移除左邊離開視窗的元素

            maxSum = max(maxSum, windowSum);
        }

        return static_cast<double>(maxSum) / k;
    }
};
```

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

class Solution {
public:
    double findMaxAverage(vector<int>& nums, int k) {
        int windowSum = 0;

        for (int i = 0; i < k; i++) {
            windowSum += nums[i];
        }

        int maxSum = windowSum;

        for (int right = k; right < nums.size(); right++) {
            windowSum += nums[right];
            windowSum -= nums[right - k];

            maxSum = max(maxSum, windowSum);

        return static_cast<double>(maxSum) / k;
    }
};

int main() {
    Solution sol;

    vector<int> nums = {1, 12, -5, -6, 50, 3};
    int k = 4;

    cout << sol.findMaxAverage(nums, k) << endl;

}
```

## 14. 程式逐行說明

### 14.1 建立目前視窗總和

```cpp
int windowSum = 0;
```

用來記錄：

```text
目前長度為 k 的子陣列總和
```

---

### 14.2 計算第一個視窗

```cpp
for (int i = 0; i < k; i++) {
    windowSum += nums[i];
}
```

這段會計算：

```text
nums[0] 到 nums[k-1]
```

的總和。

---

### 14.3 初始化最大總和

```cpp
int maxSum = windowSum;
```

第一個視窗本身也可能就是答案，所以先當作目前最大值。

---

### 14.4 從第 k 個元素開始滑動視窗

```cpp
for (int right = k; right < nums.size(); right++) {
```

為什麼從 `k` 開始？

因為：

```text
前 k 個元素已經形成第一個視窗
```

下一個新進入視窗的元素就是：

```text
nums[k]
```

---

### 14.5 加入右邊的新元素

```cpp
windowSum += nums[right];
```

表示新視窗比舊視窗多加入一個右側元素。

---

### 14.6 移除左邊離開視窗的元素

```cpp
windowSum -= nums[right - k];
```

這個元素剛好是：

```text
前一個視窗最左邊的元素
```

---

### 14.7 更新最大總和

```cpp
maxSum = max(maxSum, windowSum);
```

如果目前視窗總和更大，就更新答案。

---

### 14.8 回傳最大平均值

```cpp
return static_cast<double>(maxSum) / k;
```

因為：

```text
平均值 = 總和 / k
```

而且答案需要是 `double`。

---

## 15. 為什麼要轉成 double？

如果寫：

```cpp
return maxSum / k;
```

兩邊都是 `int`，會做：

```text
整數除法
```

例如：

```text
51 / 4 = 12
```

但正確答案應該是：

```text
12.75
```

所以需要寫：

```cpp
static_cast<double>(maxSum) / k;
```

讓除法變成浮點數除法。

## 17. Constraints 是否需要特別注意？

題目限制：

```text
1 <= k <= n <= 100000
-10000 <= nums[i] <= 10000
```

最大可能總和：

```text
100000 × 10000 = 1,000,000,000
```

這仍然在 32-bit `int` 範圍內：

```text
2,147,483,647
```

所以：

```cpp
int windowSum
int maxSum
```

是安全的。

## 18. 複雜度分析

假設：

```text
n = nums.length
```

### 時間複雜度

```text
O(n)
```

原因：

```text
每個元素最多被加入一次、移除一次
```

---

### 空間複雜度

```text
O(1)
```

只使用幾個固定變數：

```text
windowSum
maxSum
right
```

## 19. 暴力法 vs Sliding Window

| 方法 | 時間複雜度 | 空間複雜度 | 是否推薦 |
|---|---:|---:|---|
| 暴力法 | O(nk) | O(1) | 不推薦 |
| Sliding Window | O(n) | O(1) | 推薦 |

## 20. 面試講法

### 中文版

這題所有子陣列長度都固定為 `k`，所以最大平均值等價於最大總和。  
我使用固定長度的 sliding window。  
先計算前 `k` 個元素的總和，接著視窗每次向右移動一格，只需要加入新的右端元素，並移除離開視窗的左端元素。  
這樣可以避免每次重新加總整個區間，將時間複雜度從 `O(nk)` 優化成 `O(n)`。

---

### English Version

Since every subarray has the same fixed length `k`, maximizing the average is equivalent to maximizing the sum.  
I use a fixed-size sliding window.  
First, I compute the sum of the first `k` elements. Then, as the window moves to the right, I add the new element entering the window and subtract the element leaving the window.  
This avoids recomputing each subarray sum from scratch and improves the time complexity from `O(nk)` to `O(n)`.
