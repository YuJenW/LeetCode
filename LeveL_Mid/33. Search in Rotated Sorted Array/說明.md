# 33. Search in Rotated Sorted Array 解題說明

## 1. 題目說明

給定一個原本由小到大排序的整數陣列：

```text
nums
```

但是這個陣列可能被旋轉過。

例如原本是：

```text
[0,1,2,4,5,6,7]
```

旋轉後可能變成：

```text
[4,5,6,7,0,1,2]
```

現在給定一個數字：

```text
target
```

請在 `nums` 裡找出 `target` 的位置。

如果有找到，回傳它的 index。

如果找不到，回傳：

```text
-1
```

---

## 2. 題目範例

### Example 1

```text
Input:
nums = [4,5,6,7,0,1,2]
target = 0

Output:
4
```

因為：

```text
nums[4] = 0
```

所以回傳：

```text
4
```

---

### Example 2

```text
Input:
nums = [4,5,6,7,0,1,2]
target = 3

Output:
-1
```

因為 `3` 不在陣列中，所以回傳：

```text
-1
```

---

### Example 3

```text
Input:
nums = [1]
target = 0

Output:
-1
```

陣列中只有 `1`，沒有 `0`，所以回傳：

```text
-1
```

---

## 3. 題目重點

這題有幾個重要條件：

```text
1. nums 原本是由小到大排序
2. nums 可能被旋轉過
3. nums 裡面的值不重複
4. 要找 target 的 index
5. 時間複雜度必須是 O(log n)
```

---

## 4. target 不是旋轉幅度

這題容易誤會。

`target` 不是位移量，也不是旋轉幅度。

`target` 是：

```text
你要找的數字
```

例如：

```text
nums = [4,5,6,7,0,1,2]
target = 0
```

意思是：

```text
請在 nums 裡找數字 0 的位置
```

答案是：

```text
4
```

因為：

```text
nums[4] = 0
```

---

## 5. 為什麼不能用線性搜尋？

最簡單做法是從頭掃到尾：

```cpp
for (int i = 0; i < nums.size(); i++) {
    if (nums[i] == target) {
        return i;
    }
}
return -1;
```

這樣時間複雜度是：

```text
O(n)
```

但題目要求：

```text
O(log n)
```

所以必須使用：

```text
Binary Search 二分搜尋
```

---

## 6. 這題和普通 Binary Search 差在哪？

普通 Binary Search 的前提是：

```text
整個陣列都是由小到大排序
```

例如：

```text
[0,1,2,4,5,6,7]
```

但這題是旋轉後的排序陣列：

```text
[4,5,6,7,0,1,2]
```

整體看起來不是完全遞增。

所以不能直接用普通 binary search。

---

## 7. 這題的核心觀察

雖然整個陣列被旋轉過，但只要用 `mid` 切成兩半：

```text
left ~ mid
mid ~ right
```

其中至少有一半會是正常排序的。

---

### 例子

```text
nums = [4,5,6,7,0,1,2]
```

如果：

```text
left = 0
right = 6
mid = 3
```

則：

```text
nums[left] = 4
nums[mid] = 7
nums[right] = 2
```

左半邊：

```text
[4,5,6,7]
```

是正常排序的。

右半邊：

```text
[0,1,2]
```

也是正常排序的。

---

另一種情況：

```text
nums = [6,7,0,1,2,4,5]
```

如果：

```text
left = 0
right = 6
mid = 3
```

則：

```text
nums[left] = 6
nums[mid] = 1
nums[right] = 5
```

左半邊：

```text
[6,7,0,1]
```

不是正常排序。

右半邊：

```text
[1,2,4,5]
```

是正常排序。

---

## 8. 解題核心想法

每一輪 binary search 都做三件事：

```text
1. 檢查 nums[mid] 是否等於 target
2. 判斷哪一半是正常排序的
3. 看 target 是否落在正常排序的那一半
```

如果 target 在正常排序的那一半，就往那一半找。

如果不在，就往另一半找。

---

## 9. 怎麼判斷左半邊有沒有排序？

使用：

```cpp
if (nums[left] <= nums[mid])
```

如果成立，代表：

```text
left 到 mid 這段是正常排序的
```

也就是：

```text
nums[left] <= nums[left+1] <= ... <= nums[mid]
```

---

## 10. 左半邊排序時，怎麼判斷 target 在不在左半邊？

如果左半邊是排序好的：

```text
nums[left] ~ nums[mid]
```

那只要判斷：

```cpp
nums[left] <= target && target < nums[mid]
```

如果成立，代表 target 在左半邊。

所以搜尋範圍改成：

```cpp
right = mid - 1;
```

如果不成立，代表 target 不在左半邊。

所以搜尋範圍改成：

```cpp
left = mid + 1;
```

---

## 11. 為什麼是 `target < nums[mid]`，不是 `target <= nums[mid]`？

因為前面已經先判斷過：

```cpp
if (nums[mid] == target)
```

如果相等，早就回傳 `mid` 了。

所以後面只需要判斷：

```cpp
target < nums[mid]
```

---

## 12. 右半邊排序時，怎麼判斷 target 在不在右半邊？

如果左半邊不是排序好的，代表右半邊一定是排序好的。

右半邊範圍是：

```text
nums[mid] ~ nums[right]
```

判斷 target 是否在右半邊：

```cpp
nums[mid] < target && target <= nums[right]
```

如果成立，往右半邊找：

```cpp
left = mid + 1;
```

否則往左半邊找：

```cpp
right = mid - 1;
```

---

## 13. 整體判斷邏輯

可以整理成：

```text
每一輪：

1. mid 是 target 嗎？
   是：回傳 mid

2. 左半邊有排序嗎？
   是：
      target 在左半邊範圍嗎？
         是：往左找
         否：往右找

   否：
      右半邊一定有排序
      target 在右半邊範圍嗎？
         是：往右找
         否：往左找
```

## 14. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    int search(vector<int>& nums, int target) {
        int left = 0;
        int right = nums.size() - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            }

            // 左半邊是正常排序
            if (nums[left] <= nums[mid]) {
                // target 在左半邊範圍內
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } 
                // target 不在左半邊，往右半邊找
                else {
                    left = mid + 1;
                }
            } 
            // 右半邊是正常排序
            else {
                // target 在右半邊範圍內
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } 
                // target 不在右半邊，往左半邊找
                else {
                    right = mid - 1;
                }
            }
        }

        return -1;
    }
};
```

## 15. 如果在本機執行，需要補 include

```cpp
#include <iostream>
#include <vector>

using namespace std;
```

完整版本：

```cpp
#include <iostream>
#include <vector>

using namespace std;

class Solution {
public:
    int search(vector<int>& nums, int target) {
        int left = 0;
        int right = nums.size() - 1;

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            }

            if (nums[left] <= nums[mid]) {
                if (nums[left] <= target && target < nums[mid]) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            } else {
                if (nums[mid] < target && target <= nums[right]) {
                    left = mid + 1;
                } else {
                    right = mid - 1;
                }
            }
        }

        return -1;
    }
};

int main() {
    Solution sol;

    vector<int> nums = {4, 5, 6, 7, 0, 1, 2};
    int target = 0;

    cout << sol.search(nums, target) << endl;

    return 0;
}
```

## 16. 程式逐行說明

### 16.1 建立左右指標

```cpp
int left = 0;
int right = nums.size() - 1;
```

`left` 是目前搜尋範圍的左邊界。

`right` 是目前搜尋範圍的右邊界。

一開始搜尋整個陣列。

---

### 16.2 Binary Search 主迴圈

```cpp
while (left <= right)
```

只要搜尋範圍還存在，就繼續找。

如果 `left > right`，代表 target 不存在。

---

### 16.3 計算 mid

```cpp
int mid = left + (right - left) / 2;
```

這是 binary search 的中間位置。

不用：

```cpp
(left + right) / 2
```

是為了避免 `left + right` 太大時 overflow。

---

### 16.4 如果找到 target

```cpp
if (nums[mid] == target) {
    return mid;
}
```

如果中間位置剛好就是 target，直接回傳 index。

---

### 16.5 判斷左半邊是否排序

```cpp
if (nums[left] <= nums[mid])
```

如果成立，代表：

```text
nums[left] 到 nums[mid] 是正常排序的
```

---

### 16.6 target 在左半邊範圍內

```cpp
if (nums[left] <= target && target < nums[mid])
```

如果成立，代表 target 應該在左半邊。

所以：

```cpp
right = mid - 1;
```

---

### 16.7 target 不在左半邊

```cpp
else {
    left = mid + 1;
}
```

左半邊排序，但 target 不在左半邊範圍內。

所以往右半邊找。

---

### 16.8 右半邊是排序好的

```cpp
else {
```

如果左半邊不是排序好的，代表右半邊一定是排序好的。

---

### 16.9 target 在右半邊範圍內

```cpp
if (nums[mid] < target && target <= nums[right])
```

如果成立，代表 target 在右半邊。

所以：

```cpp
left = mid + 1;
```

---

### 16.10 target 不在右半邊

```cpp
else {
    right = mid - 1;
}
```

右半邊排序，但 target 不在右半邊範圍內。

所以往左半邊找。

---

### 16.11 找不到 target

```cpp
return -1;
```

如果 while 結束都沒有回傳，代表 target 不存在於陣列中。

## 19. 為什麼這樣是 O(log n)？

每一輪都會把搜尋範圍砍掉一半。

例如：

```text
n = 8
```

搜尋範圍可能變化：

```text
8 -> 4 -> 2 -> 1
```

所以時間複雜度是：

```text
O(log n)
```

---

## 20. 空間複雜度

只使用：

```text
left
right
mid
```

幾個變數。

所以空間複雜度是：

```text
O(1)
```

## 21. 常見錯誤

### 錯誤 1：把 target 當成 index

錯誤想法：

```text
target 是 index
```

正確是：

```text
target 是要找的值
```

要回傳的是：

```text
target 所在的 index
```

---

### 錯誤 2：只看最大值和最小值

例如：

```text
nums = [4,5,6,7,0,1,2]
target = 3
```

整個陣列最小是 `0`，最大是 `7`。

`3` 在 `0 ~ 7` 之間。

但 `3` 不在陣列中。

所以只看最大最小值是不夠的。

---

### 錯誤 3：忘記判斷哪一半是排序好的

這題不能直接用普通 binary search。

每一輪都要先判斷：

```text
左半邊排序？
還是右半邊排序？
```

再決定往哪一邊找。

## 22. 面試講法

### 中文版

這題雖然陣列被旋轉過，但每次用 `mid` 切開時，左右兩半至少有一半是正常排序的。  
我使用 binary search。  
每一輪先判斷 `nums[mid]` 是否為 target。若不是，接著判斷左半邊是否排序。  
如果左半邊排序，就檢查 target 是否落在 `nums[left]` 到 `nums[mid]` 的範圍內。若是，就往左找，否則往右找。  
如果左半邊沒有排序，代表右半邊排序，接著用同樣方式檢查 target 是否在右半邊範圍內。  
每一輪都能排除一半搜尋範圍，因此時間複雜度是 `O(log n)`。

---

### English Version

Although the array is rotated, when we split the current range by `mid`, at least one half must still be sorted.  
I use binary search.  
At each step, I first check whether `nums[mid]` is the target. If not, I determine whether the left half is sorted.  
If the left half is sorted, I check whether the target lies within `nums[left]` and `nums[mid]`. If it does, I search the left half; otherwise, I search the right half.  
If the left half is not sorted, then the right half must be sorted, and I apply the same logic to the right half.  
Since each step eliminates half of the search space, the time complexity is `O(log n)`.

---

## 23. 一句話總結

```text
這題的核心是：旋轉陣列每次切一半，至少有一半是排序好的；利用排序好的那一半判斷 target 是否可能存在，逐步縮小搜尋範圍。
```
