# 2390. Removing Stars From a String 解題說明

## 1. 題目說明

給定一個字串：

```text
s
```

其中包含：

```text
小寫英文字母
以及星號 *
```

每遇到一個 `*`，需要執行一次刪除操作：

```text
刪除這個 * 本身
以及它左邊最近的一個非 * 字元
```

最後回傳：

```text
所有 * 都移除後的字串
```

---

## 2. 題目保證

題目特別說明：

```text
每一個 * 左邊一定都有可以刪除的字元 (The input will be generated such that the operation is always possible.)
```

所以不會出現：

```text
"*abc"
```

這種無法操作的情況。

---

## 3. 題目範例

### Example 1

```text
Input:
s = "leet**cod*e"

Output:
"lecoe"
```

---

### 操作過程

原字串：

```text
leet**cod*e
```

---

#### 第 1 個 `*`

它左邊最近的非星號字元是：

```text
t
```

刪除 `t` 和 `*`：

```text
lee*cod*e
```

---

#### 第 2 個 `*`

它左邊最近的非星號字元是：

```text
e
```

刪除 `e` 和 `*`：

```text
lecod*e
```

---

#### 第 3 個 `*`

它左邊最近的非星號字元是：

```text
d
```

刪除 `d` 和 `*`：

```text
lecoe
```

---

最後答案：

```text
"lecoe"
```

---

### Example 2

```text
Input:
s = "erase*****"

Output:
""
```

因為：

```text
erase
```

會被後面的五個 `*` 全部刪除。

---

## 4. 這題的本質

這題本質上是在做：

```text
遇到一般字元就保存
遇到 * 就刪掉最近保存的字元
```

這個行為和資料結構：

```text
Stack，堆疊
```

完全一致。

---

## 5. 為什麼適合 Stack？

Stack 的規則是：

```text
Last In, First Out
後進先出
```

也就是：

```text
最後放進去的元素，最先被拿出來
```

---

### 對應到這題

當遇到字母時：

```text
把字母放進 stack
```

當遇到 `*` 時：

```text
刪除 stack 最上方的字母
```

因為 stack 最上方的字母，正好就是：

```text
* 左邊最近的尚未被刪除字元
```

---

## 6. 需要的知識點

這題主要需要理解：

```text
1. Stack 堆疊
2. LIFO 後進先出
3. string 的 push_back()
4. string 的 pop_back()
5. 用字串模擬 stack
```

---

# 第一種解法：使用 string 當作 stack

## 7. 為什麼可以用 string 當 stack？

C++ 的 `string` 提供：

```cpp
push_back()
pop_back()
```

可以很方便地：

```text
在字串尾端加入字元
從字串尾端刪除字元
```

這和 stack 的：

```text
push
pop
```

概念一樣。

---

## 8. 解題邏輯

逐一掃描 `s` 中的每個字元：

### 如果是一般字母

```text
加入結果字串 ans 的尾端
```

### 如果是 `*`

```text
刪除 ans 最後一個字元
```

---

## 9. LeetCode 可直接提交程式碼

```cpp
class Solution {
public:
    string removeStars(string s) {
        string ans;

        for (char ch : s) {
            if (ch == '*') {
                ans.pop_back();
            } else {
                ans.push_back(ch);
            }
        }

        return ans;
    }
};
```

---

## 10. 程式逐行說明

### 10.1 建立答案字串

```cpp
string ans;
```

這個 `ans` 同時扮演：

```text
答案字串
以及
stack
```

---

### 10.2 逐一讀取字元

```cpp
for (char ch : s) {
```

表示：

```text
依序把 s 中每個字元取出來
```

---

### 10.3 遇到 `*`

```cpp
if (ch == '*') {
    ans.pop_back();
}
```

`pop_back()` 會刪除：

```text
ans 最後一個字元
```

而這個最後一個字元正是：

```text
* 左邊最近的未被刪除字元
```

---

### 10.4 遇到一般字元

```cpp
else {
    ans.push_back(ch);
}
```

把目前字元加入 `ans` 尾端。

---

### 10.5 回傳答案

```cpp
return ans;
```

當所有字元都掃描完成後，`ans` 就是最終結果。

## 12. 為什麼 `pop_back()` 一定安全？

因為題目保證：

```text
The operation can always be performed.
```

也就是：

```text
每個 * 出現時，
左邊一定還有一個可刪除的字元。
```

所以：

```cpp
ans.pop_back();
```

不會對空字串執行。

---

## 13. 複雜度分析

假設：

```text
n = s.length()
```

### 時間複雜度

```text
O(n)
```

原因：

```text
每個字元只處理一次
```

---

### 空間複雜度

```text
O(n)
```

最壞情況下：

```text
s 中沒有任何 *
```

那 `ans` 會存下所有字元。

# 第二種解法：使用 stack<char>

## 14. 另一種寫法

也可以明確使用：

```cpp
stack<char>
```

---

## 15. LeetCode 程式碼：stack 版本

```cpp
class Solution {
public:
    string removeStars(string s) {
        stack<char> st;

        for (char ch : s) {
            if (ch == '*') {
                st.pop();
            } else {
                st.push(ch);
            }
        }

        string ans;

        while (!st.empty()) {
            ans.push_back(st.top());
            st.pop();
        }

        reverse(ans.begin(), ans.end());

        return ans;
    }
};
```

---

## 16. stack 版本為什麼比較麻煩？

因為 `stack<char>` 取出的順序是：

```text
最後放入的先取出
```

如果直接取出來，順序會反過來。

---

### 例子

stack 裡面概念上是：

```text
l -> e -> c -> o -> e
```

但 `top()` 會先拿到：

```text
e
o
c
e
l
```

組成：

```text
"eocel"
```

所以最後還要：

```cpp
reverse(ans.begin(), ans.end());
```

才能得到：

```text
"lecoe"
```

---

## 17. string 當 stack vs stack<char>

| 方法 | 優點 | 缺點 |
|---|---|---|
| `string + push_back/pop_back` | 程式最短，直接回傳 | 本質上是用 string 模擬 stack |
| `stack<char>` | 資料結構語意明確 | 最後還要反轉答案，比較麻煩 |

---

## 18. 哪個版本比較推薦？

這題我會推薦：

```text
string 當 stack
```

原因：

```text
1. 寫法最簡潔
2. 不需要額外 reverse
3. ans 本身就是答案
4. push_back/pop_back 剛好對應 stack 操作
```

---

## 19. 面試講法

### 中文版

這題我使用 stack 的概念來處理。  
遇到一般字元時，就將它加入答案字串尾端；遇到 `*` 時，就刪除答案字串最後一個字元。  
因為最後加入的字元，正好就是 `*` 左邊最近的未刪除字元，所以可以用 `pop_back()` 直接移除。  
整體時間複雜度是 `O(n)`。

---

### English Version

I use a stack-like approach.  
For each normal character, I append it to the result string.  
When I encounter a star, I remove the last character from the result using `pop_back()`.  
The last character in the result is exactly the closest non-star character to the left of the star.  
This gives an `O(n)` solution.

---

## 20. 一句話總結

```text
這題的核心是 stack：字母加入尾端，遇到 * 就刪除尾端，最後留下的字串就是答案。
```

## 21. 進一步最佳化：不用 `push_back()` / `pop_back()`，直接在原字串上覆寫

前面的解法使用：

```cpp
ans.push_back(ch);
ans.pop_back();
```

利用 `string` 模擬 stack。

這種寫法很直觀，但會額外建立一個新的字串：

```cpp
string ans;
```

---

如果想進一步節省空間，可以改成：

> **直接在原本的字串 `s` 上修改，不另外建立 `ans`。**

---

## 22. LeetCode 可直接提交程式碼：原地覆寫版本

```cpp
class Solution {
public:
    string removeStars(string s) {
        int write = 0;

        for (char ch : s) {
            if (ch == '*') {
                write--;
            } else {
                s[write] = ch;
                write++;
            }
        }

        s.resize(write);
        return s;
    }
};
```

---

## 23. 這個版本的核心想法

這個版本使用：

```cpp
int write = 0;
```

把 `write` 當成：

```text
目前保留下來的有效字串長度
```

也可以理解成：

```text
下一個字元應該寫入的位置
```

---

## 24. 遇到一般字元時

```cpp
s[write] = ch;
write++;
```

意思是：

```text
把這個字元保留下來，
寫到目前有效字串的尾端，
然後 write 往後移一格。
```

---

### 例子

目前：

```text
write = 0
ch = 'l'
```

執行：

```cpp
s[write] = ch;
write++;
```

之後：

```text
s[0] = 'l'
write = 1
```

代表目前有效內容是：

```text
"l"
```

---

## 25. 遇到 `*` 時

```cpp
write--;
```

意思是：

```text
刪除目前有效字串最後一個字元。
```

---

### 為什麼 `write--` 就代表刪除？

假設目前有效內容是：

```text
"leet"
```

那麼：

```text
write = 4
```

因為下一個可以寫入的位置是 index 4。

---

如果遇到：

```text
*
```

執行：

```cpp
write--;
```

變成：

```text
write = 3
```

這表示：

```text
有效字串現在只保留 index 0 ~ 2
```

也就是：

```text
"lee"
```

原本的 `'t'` 雖然還暫時留在記憶體中，但之後會被覆寫或被忽略。

---

輸入：

```text

```

---

| 讀到字元 | 操作 | write | 有效內容 |
|---|---|---:|---|
| `l` | 寫入 `s[0]` | 1 | `"l"` |
| `e` | 寫入 `s[1]` | 2 | `"le"` |
| `e` | 寫入 `s[2]` | 3 | `"lee"` |
| `t` | 寫入 `s[3]` | 4 | `"leet"` |
| `*` | `write--` | 3 | `"lee"` |
| `*` | `write--` | 2 | `"le"` |
| `c` | 寫入 `s[2]` | 3 | `"lec"` |
| `o` | 寫入 `s[3]` | 4 | `"leco"` |
| `d` | 寫入 `s[4]` | 5 | `"lecod"` |
| `*` | `write--` | 4 | `"leco"` |
| `e` | 寫入 `s[4]` | 5 | `"lecoe"` |

```text
write = 5
```

代表答案有效長度為：

```text
5
```

## 27. 為什麼最後要 `resize(write)`？

處理過程中，我們只是改變：

```text
哪些位置算有效
```

但原本字串 `s` 的實際長度還沒有變短。

例如最後有效內容是：

```text
"lecoe"
```

但 `s` 原本長度仍然可能還是：

```text
11
```

所以需要：

```cpp
s.resize(write);
```

把字串真正裁切成：

```text
只保留前 write 個字元
```

---

### 例子

如果：

```text
s 前面有效內容 = "lecoe"
write = 5
```

執行：

```cpp
s.resize(5);
```

最後回傳：

```text
"lecoe"
```

```cpp
string ans;
```

會另外開一個字串來存結果。

這一版直接使用原本傳入的：


## 29. 與 `push_back()` / `pop_back()` 版本比較

### 版本一：使用新字串當 stack

```cpp
class Solution {
public:
    string removeStars(string s) {
        string ans;

        for (char ch : s) {
            if (ch == '*') {
                ans.pop_back();
            } else {
                ans.push_back(ch);
            }
        }

        return ans;
    }
};
```

---

### 版本二：直接在原字串上覆寫

```cpp
class Solution {
public:
    string removeStars(string s) {
        int write = 0;

        for (char ch : s) {
            if (ch == '*') {
                write--;
            } else {
                s[write] = ch;
                write++;
            }
        }

        s.resize(write);
        return s;
    }
};
```

---

## 30. 兩種方法優缺點

| 方法 | 優點 | 缺點 |
|---|---|---|
| `push_back()` / `pop_back()` | 最直觀，stack 概念明確 | 需要額外建立 `ans` |
| `write` 原地覆寫 | 不另外建立答案字串，額外空間更省 | 需要理解 `write` 指標代表有效長度 |

---

## 31. 為什麼原地覆寫版本通常記憶體較省？

因為前一版額外建立：

```cpp
string ans;
```

最壞情況下，如果沒有任何 `*`：

```text
ans 會存下整個 s 的內容
```

所以額外空間可能接近：

```text
O(n)
```

---

原地覆寫版本則只使用：

```cpp
int write;
```

並直接修改原字串 `s`。

因此額外空間可以視為：

```text
O(1)
```

---

## 32. 兩種版本複雜度比較

| 方法 | 時間複雜度 | 額外空間複雜度 |
|---|---:|---:|
| `push_back()` / `pop_back()` | `O(n)` | `O(n)` |
| 原地覆寫 `write` | `O(n)` | `O(1)` |

---

## 33. 哪個版本比較推薦？

### 第一次理解題目時

推薦：

```text
push_back() / pop_back()
```

因為它非常符合 stack 的概念。

---

### 想進一步優化記憶體時

推薦：

```text

```

因為它：

```text

2. 只使用一個整數變數
3. 邏輯仍然是 stack，只是把 stack 指標改成 write
```

## 34. 面試講法

### 中文版

除了用 `string` 搭配 `push_back()` 和 `pop_back()` 模擬 stack，也可以直接在原字串上做原地覆寫。  
我使用 `write` 代表目前有效字串的長度。  
遇到一般字元時，就寫入 `s[write]`，再讓 `write++`；遇到 `*` 時，就讓 `write--`，表示刪除前一個有效字元。  
最後使用 `resize(write)` 裁掉後方無效內容。  
這樣可以將額外空間從 `O(n)` 降到 `O(1)`。

---

### English Version

Besides using a separate string with `push_back()` and `pop_back()` as a stack, we can overwrite the original string in place.  
I use `write` to represent the current length of the valid result.  
For a normal character, I write it into `s[write]` and increment `write`.  
For a star, I decrement `write`, which effectively removes the previous valid character.  
Finally, I resize the string to `write`.  
This reduces the extra space from `O(n)` to `O(1)`.

---

## 35. 一句話總結

```text
不用 push_back() / pop_back() 時，可以用 write 指標直接覆寫原字串：字母寫入，星號回退，最後 resize 保留有效結果。
```
